# Implementation Diary: Task 0.4 - Add Tauri v2 Shell and Rust Backend

## Date: 2026-03-10

## Summary

Initialized Tauri v2 within the existing `src-tauri/` directory structure, preserving all 17 placeholder
README.md files. Created the Rust project (Cargo.toml, build.rs, main.rs, lib.rs), Tauri configuration
(tauri.conf.json), default capability, placeholder icons, and mod.rs files for all module directories.

## Approach

Manually created all Tauri v2 files rather than running `npx tauri init`, because the `src-tauri/`
directory already existed with a curated subdirectory layout (commands/, core/*, events/, models/,
state/, capabilities/, tests/) containing placeholder README.md files. Running `tauri init` would
have overwritten or conflicted with this structure.

## Key Decisions

- **Tauri v2.10.3** (latest stable) used for the Rust crate; `@tauri-apps/cli@2.10.1` and
  `@tauri-apps/api@2.10.1` for npm packages.
- **Library crate** (`vigil_lib`) with `crate-type = ["lib", "cdylib", "staticlib"]` to support
  Tauri's mobile and desktop targets.
- **Module tree**: `lib.rs` declares `pub mod` for all six top-level modules (commands, core,
  events, models, state) matching the architecture doc. `core/mod.rs` re-exports all eight
  sub-modules (fs, index, content, search, graph, git, links, plugins).
- **Placeholder icons** generated as minimal RGBA PNGs (32x32, 128x128, 256x256) plus .ico and
  .icns using Python. These are solid dark (#1E1E2E) squares to be replaced with real branding.
- **No devtools in release**: `open_devtools()` is gated behind `#[cfg(debug_assertions)]`.
- **Release profile** optimized: `lto = true`, `codegen-units = 1`, `opt-level = "s"`, `strip = true`.
- **System Rust** (Arch Linux packages, no rustup) confirmed working with `cargo check` and `cargo clippy`.
- **Identifier**: `dev.vigil.app` used in tauri.conf.json.
