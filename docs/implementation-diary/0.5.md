# Implementation Diary: Task 0.5 - Configure Static SPA Output for Tauri

## Date: 2026-03-10

## Summary

Switched SvelteKit from `adapter-auto` to `adapter-static` so that `npm run build` produces a
fully static SPA in the `build/` directory, which Tauri v2's `frontendDist: "../build"` already
points to.

## Approach

1. Installed `@sveltejs/adapter-static` as a dev dependency.
2. Updated `svelte.config.js` to import from `@sveltejs/adapter-static` instead of `adapter-auto`.
3. Created `src/routes/+layout.ts` exporting `prerender = true` and `ssr = false` to enable
   full static prerendering with client-side-only rendering (SPA mode for Tauri).
4. Verified `tauri.conf.json` already had `frontendDist: "../build"` (no change needed).
5. Ran `npm run build` successfully; confirmed `build/index.html` exists.

## Key Decisions

- **adapter-static with no options**: The default output directory is `build`, which matches
  the Tauri config's `frontendDist: "../build"`. No `pages`/`assets`/`fallback` overrides needed.
- **`ssr = false`**: Required for Tauri since there is no server; all rendering happens client-side.
- **`prerender = true`**: Enables static generation of all routes at build time.
- **adapter-auto kept in devDependencies**: Not removed, as it has no runtime cost and removing
  it is cosmetic. Can be cleaned up later if desired.

## Verification Results

- `npm run build`: succeeded, output written to `build/`
- `build/index.html`: exists
- `build/_app/`: exists with immutable chunks
- `tauri.conf.json` `frontendDist`: already `"../build"` (unchanged)
- Vite warning about circular dependency in Svelte internals (reexported `tick`) is a known
  upstream issue and does not affect functionality.
