# Task 0.6: Install Tailwind and baseline design tokens

## Date: 2026-03-10

## Summary

Installed Tailwind CSS v4 with the `@tailwindcss/vite` plugin approach and established
baseline design tokens for Vigil's dark-first theme.

## Changes

### `package.json` / `package-lock.json`
- Added `tailwindcss` and `@tailwindcss/vite` as devDependencies.

### `vite.config.ts`
- Imported `@tailwindcss/vite` plugin.
- Added `tailwindcss()` to the Vite plugins array (before `sveltekit()`).

### `src/app.css` (new)
- Tailwind v4 entry: `@import "tailwindcss"`.
- `@theme` block defining baseline design tokens:
  - Surface palette: `surface-base`, `surface-raised`, `surface-overlay`, `surface-border`.
  - Accent scale: `accent`, `accent-muted`, `accent-strong`.
  - Text hierarchy: `text-primary`, `text-secondary`, `text-muted`.
  - Semantic colors: `success`, `warning`, `error`, `info`.
  - Typography: `font-sans` (Inter), `font-mono` (JetBrains Mono).
  - Border radii: `radius-sm/md/lg`.
  - Z-index layers: `z-base/raised/overlay/modal/toast`.
- Global baseline resets for `html` and `body`.

### `src/routes/+layout.svelte`
- Added `import '../app.css'` in the script block.

### `src/routes/+page.svelte`
- Wrapped content in a `<main>` with Tailwind utility classes (`flex`, `min-h-screen`,
  `items-center`, `justify-center`, `text-4xl`, `font-bold`, `text-accent`,
  `text-text-secondary`) to verify Tailwind processing.

## Verification

- `npm run build` succeeds; output CSS (~18.77 kB) contains Tailwind utility classes.
- `npm run dev` starts without errors.
- Build output confirmed to include `min-h-screen`, `flex`, `items-center`, etc.

## Decisions

- Used Tailwind v4 `@tailwindcss/vite` plugin (not the older PostCSS approach) per task spec.
- Design tokens use `@theme` directive so they generate Tailwind utility classes automatically
  (e.g., `text-accent`, `bg-surface-base`).
- Chose dark-first palette values aligned with modern code editor aesthetics (low-contrast
  surfaces, blue accent, neutral text hierarchy).
