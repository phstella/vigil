# 1.11 -- Backend Integration Tests & Performance Instrumentation

## What was done

### Cross-service integration tests (`tests/integration_workflows.rs`)

Added 9 end-to-end workflow tests that exercise multiple services in
composition, using temporary workspaces with realistic markdown content:

1. **Open -> Index -> Fuzzy Find**: Opens workspace, builds index, verifies
   fuzzy search returns correct results for various queries including empty
   query (recent files).

2. **Write -> Incremental Index -> Search**: Writes a new file through
   `WorkspaceFs`, triggers incremental index update, verifies both content
   search and fuzzy find discover the new file.

3. **Tag Index Correct Counts**: Builds file and tag indices, verifies
   frontmatter tags, inline `#tags`, tag counts, `get_files_by_tag`, and
   `get_file_tags` all return correct data.

4. **Link Graph Backlinks Resolve**: Builds file index and link graph, verifies
   wikilink backlinks resolve correctly across multiple notes, outgoing links
   are tracked, and graph visualization includes all nodes/edges.

5. **Workspace Status Assembly**: Simulates the status bar data collection
   pipeline: open workspace, build indices, collect metrics, verify counts.

6. **File Lifecycle Index Consistency**: Full create-modify-delete cycle through
   `WorkspaceFs` with incremental index updates at each step, verifying search
   and metadata remain consistent.

7. **Tag/Link Coherence After Edit**: Modifies a note's tags and links, rebuilds
   tag index and link graph, verifies stale data is cleared and new data
   appears.

8. **Content Search Across Subdirectories**: Creates files in multiple
   subdirectories, verifies content search finds results across all of them.

9. **Git Integration With Workspace Operations**: Creates a git repo, commits,
   modifies through `WorkspaceFs`, verifies git status/hunks detect changes and
   content search finds new content.

### Performance instrumentation (`src/core/perf.rs`)

Added a lightweight `PerfTimer` RAII type and `time_operation` helper:

- `PerfTimer::start("label")` -- logs elapsed time on drop via `eprintln!` in
  debug builds or when `VIGIL_PERF` env var is set.
- `time_operation(|| ...)` -- returns `(result, Duration)` without logging.

Instrumented 4 hot paths with `PerfTimer`:
- `FileIndex::full_scan`
- `FuzzyFinder::fuzzy_find`
- `ContentSearcher::search_content`
- `GitService::get_hunks`

### Performance budget tests (`tests/perf_budget.rs`)

Added 9 tests that measure and print hot-path latency, comparing against
generous debug-build thresholds:

- `perf_full_scan_500_files` -- scans 500 files, asserts < 5000 ms
- `perf_fuzzy_find_latency` -- median of 5 queries over 500 files, asserts < 1600 ms
- `perf_content_search_latency` -- median of 4 queries over 500 files, asserts < 3000 ms
- `perf_tag_index_rebuild` -- 500 files, asserts < 500 ms
- `perf_link_graph_rebuild` -- 200 linked files, asserts < 5000 ms
- `perf_git_hunks_latency` -- 2000-line file with modifications, asserts < 2000 ms
- `perf_full_workflow_open_to_search` -- full pipeline for 200 files, asserts < 10000 ms
- Unit tests for `PerfTimer` and `time_operation`
