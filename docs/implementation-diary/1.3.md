### Key design decisions

1. **Path confinement**: All paths are resolved via `confined_path()` which canonicalizes existing targets and walks up to the nearest existing ancestor for new entries. Paths containing `..` are fast-rejected before any filesystem access. All resolved paths are verified to `starts_with(workspace_root)`.

2. **Binary detection**: First 8 KB of file content is scanned for null bytes. Files containing null bytes return `BINARY_FILE` error.

3. **Optimistic concurrency**: `write_file` accepts an optional `etag` (content hash). When provided, the current file's content hash is compared and `STALE_ETAG` is returned on mismatch.

4. **Sorting**: `list_dir` returns directories first, then files, both sorted case-insensitively. `.git` directories are filtered out.

5. **State management**: `AppState` wraps the workspace in `Arc<RwLock<Option<WorkspaceFs>>>` and is registered as Tauri managed state. Commands extract it via `State<'_, AppState>`.

6. **Limits**: `read_file` enforces 10 MB max. `list_dir` enforces 50,000 entry max with `truncated` flag.
