### Key design decisions

1. **Index structure**: `FileIndex` wraps `Arc<RwLock<IndexInner>>` where `IndexInner` holds two HashMaps: `files` (all entries by relative path) and `notes` (NoteMetadata for markdown files). The Arc+RwLock pattern allows the index to be cloned and shared across threads (watcher callback, Tauri commands).

2. **Full scan via `ignore` crate**: Uses `WalkBuilder` with `.gitignore` and `.ignore` support enabled. Hidden files are included but flagged (`is_hidden`). `.git` directories are filtered via `filter_entry()`.

3. **Markdown metadata extraction**: Parses YAML frontmatter for `title` and `tags` (both inline `[a, b]` and list `- a` formats). Falls back to first `# heading` for title, then filename stem. Extracts inline `#tag` patterns and outgoing links (`[[wikilinks]]` and `[text](path)` markdown links). External URLs are excluded from link extraction.

4. **Debounced file watcher**: Uses `notify::RecommendedWatcher` for OS-native filesystem events. A dedicated `vigil-fs-watcher` thread collects events via `crossbeam-channel` and coalesces them by path with a 150ms debounce window. Events that have been stable for the debounce duration are flushed as a batch.

5. **Event emission**: `vigil://index-updated` carries an array of `IndexChange` records (path, change_type, kind). `vigil://index-ready` carries file/note counts and scan duration. Both include `timestamp_ms` and `contract_version: "v1"` per the IPC contract.

6. **Workspace lifecycle**: `open_workspace` calls `clear_all()` first to tear down any previous watcher/index/workspace state, then builds fresh state. Watcher startup failure is non-fatal (logged as warning).
