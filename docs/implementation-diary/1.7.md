### Key design decisions

1. **Graceful non-git handling**: `GitService::open_repo()` returns `Option<Repository>`. When the workspace is not inside a git repository, all operations return empty results rather than errors. This prevents crashes when users open non-git workspaces.

2. **Repository discovery**: Uses `Repository::discover()` which walks up from the workspace root to find the nearest `.git` directory. This handles workspaces that are subdirectories of a git repository.

3. **Diff strategy**: Uses `diff_tree_to_workdir_with_index` to compare HEAD tree against the working directory (including staged changes). This gives a complete picture of all modifications for gutter decorations.

4. **Hunk classification**: A hunk with `old_lines == 0` is classified as Added, `new_lines == 0` as Deleted, and anything else as Modified. For pure deletions (where no new lines exist), the hunk's `start_line`/`end_line` point to the anchor line where content was removed.

5. **Path resolution**: File paths are resolved relative to the git repository workdir (not the workspace root), handling the case where the workspace is a subdirectory of the repository.

6. **Event contract compliance**: `GitHunksPayload` includes `timestamp_ms` and `contract_version: "v1"` metadata fields, matching the `vigil://git-hunks` event specification in the IPC contract.

7. **Status mapping**: git2 status bitflags are mapped to `GitFileStatus` with priority ordering: Conflicted > New > Deleted > Renamed > Modified > Clean. This ensures the most significant status is reported when multiple flags are set.
