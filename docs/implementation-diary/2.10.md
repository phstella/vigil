**Date**: 2026-03-11

## What was done

Created typed TypeScript IPC wrappers that encapsulate all Tauri v2 `invoke()` and `listen()` calls behind a single gateway module (`src/lib/ipc/tauri.ts`). All other IPC modules use this gateway so no raw `@tauri-apps/api` imports leak outside `src/lib/ipc/`.

## Design decisions

1. **Single gateway pattern**: Only `tauri.ts` imports from `@tauri-apps/api`. All other modules call `invokeCommand`/`listenEvent` so Tauri coupling is isolated to one file.
2. **Error normalization**: `invokeCommand` catches rejections, checks for the `VigilError` envelope shape, and wraps unexpected errors so callers always see a uniform error type.
3. **Alias collisions**: `src/lib/types/store.ts` already defines `DirEntry`, `GitHunk`, `SyncState`, `GitStatusEntry` for UI state. The IPC versions are re-exported from the types barrel with `Ipc` prefixes (`IpcDirEntry`, `IpcGitHunk`, etc.) to avoid collisions. IPC modules themselves import directly from `$lib/types/ipc`.
4. **Write/create_note use `request` wrapper**: The Rust commands for `write_file` and `create_note` accept a struct parameter (`WriteFileRequest` / `CreateNoteRequest`), so the TS wrappers pass `{ request: { ... } }` to match the Tauri deserialization.
5. **Optional `limit` for search commands**: `fuzzy_find` and `search_content` accept `limit: Option<u32>` in Rust, so the TS wrappers pass `null` when unspecified.
