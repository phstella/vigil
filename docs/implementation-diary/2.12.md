# 2.12 — Keyboard Shortcut Infrastructure

## What Changed

### New: `src/lib/utils/shortcuts.ts`
Created a `ShortcutRegistry` module that provides centralized keyboard shortcut management:
- Attaches a single `keydown` listener on `window` (capture phase) when shortcuts are registered.
- Normalizes combo strings (e.g. "ctrl+p") for case-insensitive, order-independent matching.
- Builds combo strings from `KeyboardEvent` properties (ctrlKey/metaKey, altKey, shiftKey + key).
- Skips non-global shortcuts when focus is in an `<input>`, `<textarea>`, `<select>`, or `contentEditable` element.
- Global shortcuts (like Ctrl+P for omnibar) fire regardless of focus context.
- Calls `preventDefault` and `stopPropagation` on matched events.
- Exports a singleton `shortcutRegistry` instance.
- `destroy()` method removes all entries and detaches the listener.

### Updated: `src/lib/utils/index.ts`
Barrel export now re-exports `shortcutRegistry`, `ShortcutEntry`, and `ShortcutRegistry` from `shortcuts.ts`.

### Updated: `src/routes/+page.svelte`
- Registers four MVP shortcuts on `onMount`, unregisters on `onDestroy`:
  - **Ctrl+P** (global): toggles omnibar via `uiStore.toggleOmnibar()`.
  - **Ctrl+S**: saves current file — calls `noteStore.markClean()` or `codeStore.markClean()` depending on which has dirty state; logs the action.
  - **Ctrl+B**: toggles sidebar via `uiStore.toggleSidebar()`.
  - **Ctrl+N**: placeholder for new note creation (logs to console).
- Removed the temporary Ctrl+P button from the status bar snippet.
- Imported `onMount`, `onDestroy`, `shortcutRegistry`, `noteStore`, and `codeStore`.
