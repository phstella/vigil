### Design decisions

- **Toggle behavior**: Clicking the active rail icon sets `activeSection` to `null`, which closes the sidebar via a `$derived` binding (`sidebarOpen = activeSection !== null`). This matches VS Code's toggle-on-reclick pattern.
- **CSS transition for collapse**: Sidebar uses `transition-[width]` with `overflow-hidden` rather than `display:none` toggling. The inner content is a fixed 260px div so text doesn't reflow during animation.
- **Exported Section type**: `PrimaryRail.svelte` exports a `Section` type alias that Sidebar and +page.svelte import, keeping the section enum in one place.
- **Inline SVG icons**: Each rail button uses a purpose-drawn SVG (folder, magnifying glass, graph nodes, tag) to avoid adding an icon library dependency.
- **Accent indicator**: Active rail button gets `bg-accent/15 text-accent` plus a 2px rounded bar on the left edge, matching the accent color from app.css tokens.
- **aria attributes**: Rail buttons use `aria-pressed` for active state; sidebar uses `aria-hidden` when collapsed.
