## What was done

Implemented the explorer panel with a recursive file tree, wired into the Sidebar component from Task 2.2. The tree renders mock workspace data with expandable folders and selectable files.

| Component | File | Purpose |

### Design decisions

- **TreeNode extends DirEntry**: Added optional `children` to the existing `DirEntry` type rather than creating a completely separate tree structure, maintaining compatibility with the store types.
- **Runes-based store**: Used Svelte 5 `$state()` for mutable state (`expandedDirs`, `selectedFile`) and plain `const` for static mock data (`tree`, `workspaceName`), satisfying the `prefer-const` lint rule.
- **Self-import for recursion**: Used `import FileTree from './FileTree.svelte'` instead of deprecated `<svelte:self>` per Svelte 5 migration guidance.
- **Depth-based indentation**: Each tree level adds 16px left padding, keeping the tree visually scannable without deeply nested DOM.
- **Chevron rotation**: Folder expand/collapse uses CSS `rotate-90` with 150ms transition for smooth animation.
- **Inline SVG icons**: Chevron, folder, and document icons are inline SVG to avoid icon library dependencies, matching the pattern established in PrimaryRail.
- **No backend wiring**: All data is mock/seed per task requirements. The store API (`toggleExpand`, `selectFile`) is designed to be easily connected to real IPC later.
