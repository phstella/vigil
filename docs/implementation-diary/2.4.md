Implemented the sidebar search panel with debounced input and mock search results. The panel renders when the PrimaryRail "Search" section is active and displays file path, line number, and matched line snippets with highlighted match text.

### Design decisions

- **Debounce via setTimeout**: Used a simple `setTimeout`/`clearTimeout` pattern (200ms) rather than pulling in a utility library. The timer reference is stored in `$state` for proper cleanup.
- **Separate searchStore**: Created a dedicated `searchStore` alongside `explorerStore` rather than nesting search state inside the explorer store, keeping concerns separated and matching the component map's distinct `SearchPanel` component.
- **$derived.by for filtered results**: Mock results are filtered reactively using `$derived.by()` against query, fileName, filePath, and lineContent fields -- case-insensitive matching.
- **Highlight helper**: A pure function `highlightMatch` splits matched text into `{ before, match, after }` segments for rendering the accent-colored match span, avoiding regex complexity.
- **Three visual states**: Empty (no query, descriptive prompt), no results (query with no matches, shows the query text), results list (count header + clickable result rows).
- **No backend wiring**: All data is mock per task requirements. The `searchStore.setQuery()` API is designed for easy replacement with `search_content` IPC calls in a later epic.
- **Inline SVG icons**: Search and clear icons are inline SVG, consistent with the pattern in PrimaryRail and ExplorerPanel.
