### Design decisions

- **Static SVG over canvas/WebGL**: Used a simple inline SVG with hardcoded coordinates rather than a force-directed physics simulation. This satisfies the shell requirement without pulling in heavy graph layout dependencies that would be replaced by real `get_note_graph` data in Epic 3.
- **Typed local data**: Defined `GraphNode` and `GraphEdge` interfaces within the component for the mock data, keeping types co-located with the only consumer.
- **Theme token usage**: Node fills use `--color-accent-muted`, strokes use `--color-accent`, edges use `--color-surface-border`, and labels use `--color-text-muted` -- all consistent with the app's design token system.
- **No store**: Unlike ExplorerPanel and SearchPanel, no store was created since the graph view is entirely static mock data with no interactive state to manage.
- **viewBox-based scaling**: The SVG uses a fixed `viewBox="0 0 240 360"` with `h-full w-full` classes so it scales proportionally within the sidebar panel regardless of sidebar width.
- **Footer info strip**: A small footer shows node/edge counts and the "connect to backend in Epic 3" placeholder text, matching the task requirement for a deferred-work indicator.
