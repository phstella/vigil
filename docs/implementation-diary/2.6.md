## Design decisions

- **Writable `$derived`**: Used Svelte 5.25+ writable derived (`$derived(clamp(initialSplit))`) so the split percentage tracks the prop but can be overridden by drag interaction. Avoids the `$state` + `$effect` anti-pattern flagged by `svelte/prefer-writable-derived`.
- **Pointer capture**: The divider captures the pointer on `pointerdown` so drag continues smoothly even if the cursor leaves the divider element.
- **CSS flex sizing**: Panel sizes use `calc(N% - 2px)` to account for the 4px divider, keeping layout in CSS without JS measurement loops.
- **Keyboard accessibility**: Focused separator supports Arrow keys (1% step) and Shift+Arrow (5% step) for keyboard-only resizing.
- **ARIA**: Divider uses `role="separator"` with `aria-valuenow/min/max` and `aria-orientation`, making it a focusable separator per WAI-ARIA spec. Svelte a11y false-positive warnings suppressed with `svelte-ignore` comments.
