# Task 2.7: Note Pane and Code Pane Skeletons

**Date**: 2026-03-11
**Status**: DONE
**Branch**: epic0-claude

## What was done

Implemented skeleton editor components for the note pane (markdown) and code pane, with an EditorRouter that dispatches to the correct editor based on file extension. Local stores provide per-editor state management.

### Components created

| Component | File | Purpose |
|---|---|---|
| `EditorRouter.svelte` | `src/lib/features/editor/EditorRouter.svelte` | Routes to NoteEditor for `.md` files, CodeEditor for all others. Shows "Open a file to start editing" empty state when no file is selected. Accepts `filePath` and `content` props. |
| `NoteEditor.svelte` | `src/lib/features/editor/NoteEditor.svelte` | Skeleton markdown editor. Displays file path in a tab header with a document icon, a monospace textarea for content (placeholder for future Tiptap), and a dirty-state dot indicator. Syncs props into the note store via `$effect`. |
| `CodeEditor.svelte` | `src/lib/features/editor/CodeEditor.svelte` | Skeleton code editor. Displays file path and detected language badge in a tab header, a line-number gutter that scrolls in sync with the textarea, and a dirty-state indicator. Syncs props into the code store via `$effect`. |

### Stores created

| Store | File | Purpose |
|---|---|---|
| `noteStore` | `src/lib/features/editor/note-store.ts` | Local Svelte 5 rune-based store for note editor state: `filePath`, `content`, `isDirty`. Methods: `load`, `updateContent`, `markClean`, `reset`. |
| `codeStore` | `src/lib/features/editor/code-store.ts` | Local rune-based store for code editor state: `filePath`, `content`, `language`, `isDirty`. Methods: `load`, `updateContent`, `markClean`, `reset`. Also exports `detectLanguage(path)` utility mapping file extensions to language identifiers. |

## Design decisions

- **Store pattern**: Follows the existing rune-based store convention established by `explorer-store.ts` -- a `createXxxStore()` function using `$state` with getter accessors and mutation methods. No `writable()` from `svelte/store`; these are Svelte 5 fine-grained reactive objects.
- **Global vs local stores**: The existing `editorStore` (Svelte 4 writable) serves as the global source of truth for which file is open. The new `noteStore`/`codeStore` are local feature-level stores that receive data via prop-driven `$effect` syncing. This keeps the feature self-contained while respecting the existing architecture.
- **EditorRouter wiring**: The center pane receives markdown files; the right code pane receives non-markdown files. Both use EditorRouter, which shows a friendly empty state when its `filePath` is null. This means both panes are always rendered but only one shows active content at a time.
- **Line number gutter**: Uses `Array.from()` with a keyed `{#each}` block to satisfy the `svelte/require-each-key` lint rule. Gutter scroll is synced to textarea scroll via an `onscroll` handler.
- **Language detection**: A static extension-to-language map in `code-store.ts` covers common file types (TypeScript, JavaScript, Rust, Python, etc.). Falls back to `'plaintext'` for unknown extensions.
- **No Monaco/Tiptap**: Components use plain `<textarea>` elements as explicitly scoped. These are clearly marked as placeholders for future rich editor integration.
