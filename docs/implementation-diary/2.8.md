# Task 2.8: Floating Omnibar UI

**Date**: 2026-03-11
**Status**: DONE

## What was done

Implemented a floating omnibar overlay for fuzzy file search with keyboard navigation, mock results, and backdrop dismiss. The omnibar is conditionally rendered based on the existing `uiStore.omnibarOpen` state.

### Components created

| Component | File | Purpose |
|---|---|---|
| `Omnibar.svelte` | `src/lib/features/omnibar/Omnibar.svelte` | Floating modal overlay centered near top of viewport. Contains a search input with magnifier icon, a filtered results list, and full keyboard navigation (ArrowUp/Down, Enter, Escape). Auto-focuses input on mount. Click-on-backdrop closes. Emits `onclose` and `onselect` callbacks. |
| `OmnibarItem.svelte` | `src/lib/features/omnibar/OmnibarItem.svelte` | Single result row displaying a file-type icon (emoji based on extension), file name, and workspace-relative path. Highlights with accent background when selected. Clickable to select. Uses `role="option"` with `aria-selected` for accessibility. |

| Store | File | Purpose |
|---|---|---|

## Design decisions

- **Overlay placement**: The omnibar renders outside the `<AppShell>` component tree so it can use `fixed` positioning with `z-modal` (200) to float above all workspace content without being clipped by `overflow-hidden` containers.
- **Store pattern**: Follows the rune-based store convention from `note-store.ts` and `code-store.ts` -- a `createOmnibarStore()` function using `$state` and `$derived` with getter accessors.
- **Mock data**: 10 static file entries covering markdown, TypeScript, and CSS files. Filtering uses case-insensitive substring matching against both name and path. This will be replaced by the `fuzzy_find` IPC command in a later ticket.
- **Keyboard handling**: `onkeydown` is placed on the outer `role="dialog"` element (with `tabindex="-1"` for a11y compliance). ArrowUp/Down wrap around the list. Enter dispatches `onselect` then `onclose`. Escape dispatches `onclose` only.
- **Cleanup on close**: An `$effect` teardown calls `omnibarStore.reset()` when the component is destroyed, ensuring clean state on next open.
- **Backdrop click**: Uses `onmousedown` (not `onclick`) on the outer div, checking `e.target === e.currentTarget` to distinguish backdrop clicks from clicks inside the panel.
- **File icons**: Simple emoji-based icons (`memo` for .md, `page` for .ts/.js, `palette` for .css, `folder` for others). These are placeholders until a proper icon system is integrated.
- **Temporary toggle**: A small "Ctrl+P" text button positioned absolutely in the status bar area. This is explicitly temporary and will be removed when the global keyboard shortcut system (ticket 2.12) is implemented.
