## Design decisions

- **Store pattern**: Uses `svelte/store` writable (matching `editorStore` convention) rather than rune-based stores. The status data comes from a single IPC call (`workspace_status`) and event channel (`vigil://status-updated`), so a simple writable with `updateStatus`/`patch` methods is sufficient.
- **Design tokens**: The task specified `bg-surface-panel` which does not exist in the theme. Used `bg-surface-raised` instead as it provides subtle elevation above the base surface, consistent with other panel surfaces (sidebar, editor headers).
- **Git branch icon**: Inline SVG based on the Octicons git-branch icon for a lightweight, dependency-free approach.
- **Sync state display**: Maps each `SyncState` value to both a human-readable label and a unicode symbol (checkmark, arrows, etc.) for compact visual feedback.
- **Mock data only**: No IPC calls or event listeners. The store is initialized with hardcoded mock data as specified. The `updateStatus` method is ready for future wiring to `workspace_status` command and `vigil://status-updated` events.
