## 2026-03-12T00:15 -- Handoff Notes

- All 5 IPC event channels (`index-updated`, `index-ready`, `git-hunks`, `status-updated`, `fs-renamed`) are wired from backend to frontend stores.
- Explorer tree invalidation uses `markDirDirty` which collapses affected directories so the next expand triggers a fresh `list_dir` call.
- Editor conflict detection: open files changed on disk are flagged in `editorStore.conflictFiles`. The `ConflictBanner` component can consume this to show a reload prompt.
- Deleted files that are open in the editor are automatically closed.
- Renamed files update editor tabs, explorer tree entries, and git hunk cache in-place.
- `cargo test` was not run because Rust toolchain is not available in this environment. No Rust files were modified so this is safe.
- The search index cache (omnibar) will automatically reflect changes because the backend re-indexes on file changes; subsequent `fuzzy_find` calls return updated results.
