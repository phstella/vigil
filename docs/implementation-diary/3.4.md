# Implementation Diary: Task 3.4 -- File-Type Routing (.md note flow vs code flow)

## 2026-03-12T00:00 -- Initial Plan

**Goal:** Implement file-type routing so `.md` files open in the center note pane (NoteEditor) and code files open in the right code pane (CodeEditor).

**Current state:**
- EditorRouter.svelte exists with basic `.md` vs code branching but renders a single component -- it doesn't distinguish between center and right pane placement.
- `+page.svelte` already has partial dual-pane logic: it renders two EditorRouter instances, one for center (md only) and one for right panel (code only).
- The explorer store's `selectFile()` only updates local selection -- it does not trigger file opening in the editor.
- There is no file-type classification utility.

**Plan (5 steps):**
1. Create `src/lib/utils/file-routing.ts` -- file-type classification utility (isMarkdown, isCodeFile, getFileCategory).
2. Enhance `src/lib/stores/editor.ts` -- add separate `noteFile`/`codeFile` tracking and an `openFileByType()` method that routes based on extension.
3. Wire explorer selection to editor open flow in `src/lib/features/explorer/explorer-store.ts`.
4. Update `EditorRouter.svelte` to accept a `pane` prop (note vs code) and render accordingly.
5. Update `+page.svelte` to use the new dual-pane routing with proper right panel visibility toggling.

## 2026-03-12T00:10 -- Implementation Complete

All 5 steps implemented:

1. **Created `src/lib/utils/file-routing.ts`** -- File-type classification utility with `isMarkdownFile()`, `isCodeFile()`, `getEditorPane()`, and `getFileExtension()`. Supports `.md`, `.markdown`, `.mdx` as note file extensions.

2. **Enhanced `src/lib/stores/editor.ts`** -- Added `noteFile`, `noteContent`, `codeFile`, `codeContent`, `codeLanguage` fields to `EditorState`. The `openFile()` method now routes to the correct pane fields based on file type. Updated `closeFile()` to clear the appropriate pane.

3. **Updated `src/lib/types/store.ts`** -- Added new dual-pane fields to `EditorState` interface.

4. **Updated `EditorRouter.svelte`** -- Added `pane` prop (`'note'` | `'code'` | `'auto'`). Each mode only renders the appropriate editor. Empty-state messages are pane-specific.

5. **Updated `+page.svelte`** -- Center pane uses `EditorRouter` with `pane="note"` fed by `editorState.noteFile`/`noteContent`. Right pane uses `pane="code"` fed by `editorState.codeFile`/`codeContent`. Right panel auto-shows when a code file is opened.

6. **Wired explorer -> editor** -- `explorer-store.ts` `selectFile()` now calls `editorStore.openFile()` with mock content and auto-shows the right panel for code files. Added `openRightPanel()`/`closeRightPanel()` to `ui.ts`.

7. **Updated barrel files** -- `src/lib/utils/index.ts` exports all file-routing utilities.
