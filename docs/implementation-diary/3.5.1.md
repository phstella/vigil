# Implementation Diary: Task 3.5.1 -- Remove Default Split Layout

## 2026-03-18T00:00 -- Initial Plan

**Goal:** Remove always-on code split behavior from the default workspace; keep one adaptive main editor surface.

**Current State Analysis:**
- `+page.svelte` already derives `sideBySideMode` from `uiState.rightPanelOpen` (default `false`) and passes it to `WorkspaceGrid` as `splitEnabled={sideBySideMode}`.
- `+page.svelte` already has single-pane adaptive routing: when `sideBySideMode` is false, it renders a single `EditorRouter` with `pane="auto"`, which routes `.md` to NoteEditor and non-md to CodeEditor.
- `EditorRouter.svelte` already supports `pane="auto"` mode that adaptively picks the correct editor surface.
- `uiStore` initial state has `rightPanelOpen: false`.
- **Issue:** `WorkspaceGrid.svelte` declares `splitEnabled = true` as the default prop value. Although `+page.svelte` overrides this, the component contract itself defaults to split-on, which contradicts the acceptance criteria.
- `keyboard-map.md` documents `Ctrl+\` for toggling side-by-side but does not explicitly note that single-pane is the default mode.

**Plan (3 steps):**
1. Change `WorkspaceGrid.svelte` default `splitEnabled` from `true` to `false`.
2. Update `keyboard-map.md` to clarify that single-pane adaptive routing is the default and split is user-initiated.
3. Run all required checks and verify acceptance criteria.

## 2026-03-18T00:01 -- Implementation Complete

**Changes made:**

1. `src/lib/components/layout/WorkspaceGrid.svelte`: Changed default `splitEnabled` prop from `true` to `false`. This ensures the component itself defaults to single-pane layout, matching the acceptance criteria that split mode is not auto-enabled.

2. `docs/specs/keyboard-map.md`: Clarified `Ctrl+\` entry to note that default is off (single adaptive pane), making the spec explicit about the default layout mode.
