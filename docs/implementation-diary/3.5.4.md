# Implementation Diary: Task 3.5.4 -- Remove Explorer Metadata Blocks

## 2026-03-18T00:00 -- Initial Plan

**Goal:** Remove non-essential notes/files metadata blocks from the Explorer panel (Notes count card + Collections summary rows).

**Current state:**
- `explorer-store.svelte.ts` contains:
  - `CollectionSummary` interface
  - `summarizeNode()` helper function
  - `notesCount` / `filesCount` reactive state variables
  - `collections` derived state (computes per-directory file/note counts)
  - `derivedNotesCount` derived state
  - Getters exposing `notesCount`, `filesCount`, `collections`
  - `loadWorkspaceRoot()` accepts `rootNotesCount` and `rootFilesCount` params
  - `reset()` zeroes `notesCount` and `filesCount`
- `ExplorerPanel.svelte` does NOT currently render any metadata cards or collection rows in the template (they exist only as store state, not yet wired to UI).
- `workspace-lifecycle.ts` calls `loadWorkspaceRoot(name, notes_count, files_count)` -- this file is outside allowed scope, so the `loadWorkspaceRoot` signature must remain compatible.

**Plan (4 steps):**
1. Remove `CollectionSummary` interface, `summarizeNode` helper, `collections` derived, `derivedNotesCount` derived, `notesCount`/`filesCount` state and getters from the store.
2. Keep `loadWorkspaceRoot` signature accepting the count params for backward compat but ignore them internally.
3. Remove `notesCount`/`filesCount`/`collections` from `ExplorerState` interface.
4. Verify ExplorerPanel.svelte needs no template changes (it already doesn't render metadata blocks).
5. Run all required checks.

## 2026-03-18T00:01 -- Implementation

**Changes to `explorer-store.svelte.ts`:**
- Removed `CollectionSummary` interface (lines 25-31).
- Removed `summarizeNode()` helper function (lines 36-53).
- Removed `notesCount`, `filesCount` state variables.
- Removed `collections` derived state and `derivedNotesCount` derived state.
- Removed getters for `notesCount`, `filesCount`, `collections`.

- Made `loadWorkspaceRoot` count params optional and prefixed with underscore (backward compatible).
- Removed `notesCount = 0` and `filesCount = 0` from `reset()`.

**No changes to `ExplorerPanel.svelte`:**
- Template already only renders workspace name, loading/error/empty states, and FileTree.
- No metadata cards or collection summary rows were present in the template.
