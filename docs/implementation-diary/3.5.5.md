# Implementation Diary: Task 3.5.5 - Render Mermaid Charts in Markdown Notes

## 2026-03-18T00:00 - Initial Plan

**Goal:** Render Mermaid diagrams for fenced `mermaid` code blocks in markdown preview mode.

**Plan (6 steps):**
1. Install `mermaid` npm package as a production dependency.
2. Create `MermaidBlock.svelte` component under `src/lib/features/preview/` that:
   - Accepts raw mermaid definition text
   - Uses mermaid's `render()` API to produce SVG
   - Displays SVG on success, or a styled error fallback on invalid input
   - Uses `sandbox` security level (not `loose`)
3. Modify `markdown.ts` to detect fenced `mermaid` blocks and emit a
   `<div class="md-mermaid" data-mermaid-source="...">` marker instead of the
   normal `<pre><code>` output. This keeps the markdown-to-HTML conversion
   synchronous while allowing the preview component to post-process mermaid blocks.
4. Modify `InlinePreview.svelte` to scan rendered HTML for `.md-mermaid` divs
   after mount/update, and replace them with live MermaidBlock renders.
5. Verify QA-013 coverage in `docs/qa/test-matrix.md`.
6. Run all CI-equivalent checks and fix any issues.

**Key decisions:**
- Mermaid is imported as a local npm dependency (no CDN/remote).
- Security level set to `strict` (default) -- never `loose`.
- Raw markdown source is never modified; mermaid rendering is preview-only.
- Error fallback shows the raw mermaid source with an error message overlay.

## 2026-03-18T00:01 - Implementation Steps

### Step 1: npm install mermaid
- Installed `mermaid@^11.13.0` as production dependency.
- 328 transitive packages added.

### Step 2: markdown.ts -- mermaid block detection
- Modified the fenced code block parser in `markdownToHtml()`.
- When `lang.toLowerCase() === 'mermaid'`, the raw source is base64-encoded
  (via `btoa(unescape(encodeURIComponent(...)))`) and emitted as a
  `<div class="md-mermaid" data-mermaid-source="...">` marker div.
- Non-mermaid fenced blocks are unchanged.
- Base64 encoding avoids HTML escaping issues with quotes/special chars
  inside mermaid definitions.

### Step 3: MermaidBlock.svelte -- diagram renderer component
- Created `src/lib/features/preview/MermaidBlock.svelte`.
- Accepts `source: string` prop.
- Uses dynamic `import('mermaid')` for code-splitting (mermaid is ~3.8MB).
- Initializes mermaid with `securityLevel: 'strict'`, `theme: 'dark'`,
  `startOnLoad: false`, `logLevel: 4` (ERROR only).
- On success: injects rendered SVG into a container div.
- On failure: displays a styled error fallback with error message and raw source.
- Cleans up orphaned mermaid error DOM elements.

### Step 4: InlinePreview.svelte -- mermaid mounting
- Added `$effect` that runs after each HTML render.
- Uses `requestAnimationFrame` to wait for `{@html}` DOM update.
- Scans for `.md-mermaid` divs, decodes base64 source, and mounts
  `MermaidBlock` instances via Svelte 5 `mount()` API.
- Tracks mounted blocks for cleanup on re-render/destroy via `unmount()`.

### Step 5: Barrel export + QA
- Updated `src/lib/features/preview/index.ts` to export `MermaidBlock`.
- Verified QA-013 already exists in `docs/qa/test-matrix.md`.
