# Implementation Diary: Task 3.5 -- Connect omnibar to fuzzy file search

## 2026-03-12T00:00 -- Initial Plan

**Goal:** Replace mock omnibar data with live IPC-backed fuzzy file search results and wire Enter to open the selected file.

**Plan:**
1. Rewrite `omnibar-store.ts` to call `fuzzyFind()` IPC instead of filtering mock data. Add debounce (80ms) to meet performance budget. Map `FuzzyMatch` responses to `OmnibarResult` items.
2. Update `OmnibarItem.svelte` to render `matched_indices` highlighting and a score badge per the UI component map spec.
3. Wire `handleOmnibarSelect` in `+page.svelte` to open the selected file via `readFile` IPC + `editorStore.openFileRouted` with proper file-type routing.
4. Run all required checks (check, lint, build, cargo test).
5. Fix any failures and finalize diary.

## 2026-03-12T00:01 -- Implementation Complete

### Changes made:

**`src/lib/features/omnibar/omnibar-store.ts`** -- Full rewrite:
- Removed mock `MOCK_FILES` array and local string filtering.
- Now calls `fuzzyFind()` from `$lib/ipc/search` for every query.
- Added 80ms debounce for non-empty queries to meet performance budget.
- Empty queries fire immediately (returns recent files per IPC contract).
- Added monotonic `requestId` to discard stale responses.
- Added `isLoading`, `error`, and `initialize()` for lifecycle management.
- `OmnibarResult` now includes `score`, `matchedIndices`, `display`, and `kind`.

**`src/lib/features/omnibar/OmnibarItem.svelte`** -- Enhanced rendering:
- Added `matched_indices` highlighting: characters that matched the fuzzy query render in accent color.
- Added score badge (small pill showing numeric score).
- Extended `fileIcon()` to handle more extensions and directory kind.

**`src/lib/features/omnibar/Omnibar.svelte`** -- Lifecycle improvements:
- Calls `omnibarStore.initialize()` on mount to load recent files.
- Shows loading indicator ("Searching...") during IPC calls.
- Shows error state for failed searches.
- Separates empty-results from loading state.

**`src/routes/+page.svelte`** -- File open wiring:
- `handleOmnibarSelect` now calls `readFile()` IPC, then `editorStore.openFileRouted()`.
- Imported `readFile`, `isMarkdownFile`, and `detectLanguage` for proper file-type routing.

### Checks run:
- `npm run check`: PASS (0 errors, 0 warnings)
- `npm run lint`: PASS (fixed `svelte/require-each-key` lint error by adding key to `{#each}`)
- `npm run build`: PASS (built successfully)
- `cargo test`: NOT RUN (Rust toolchain not available in this environment)
- `rg` search: Verified all omnibar/fuzzyFind/fuzzy_find/Ctrl+P references are consistent across frontend and backend.

### Decisions:
- Debounce set to 80ms to align with the <=80ms first-result-render budget.
- Empty queries fire immediately without debounce since the backend supports returning recent files.
- Stale response discard uses a monotonic request counter rather than AbortController, since Tauri invoke does not support abort signals.
