# Task 3.6 – Connect omnibar/content mode to phrase/snippet search

## 2026-03-12T00:00 – Initial Plan

**Goal:** Connect the omnibar to the backend `search_content` IPC command, adding a content search mode alongside the existing file search mode.

**Steps:**
1. Extend `UiState` / `uiStore` to track omnibar mode (`file` | `content`).
2. Extend `omnibar-store.ts` with content search mode, calling `searchContent` IPC, and a unified result type that handles both fuzzy file matches and content matches.
3. Update `Omnibar.svelte` to show mode tabs (File / Content), switch modes, and route input to the correct search backend.
4. Update `OmnibarItem.svelte` to display content match details (line number, preview with highlighted match text).
5. Register `Ctrl+Shift+F` shortcut in `+page.svelte` to open omnibar directly in content mode.
6. Run all checks (npm run check, npm run lint, npm run build, cargo test).

**Constraints:**
- Performance budget: content search result render <= 150 ms median.
- Debounce content search at 150 ms (heavier than file search).
- Do not modify Rust backend files (scope restriction).

## 2026-03-12T00:10 – Implementation Complete

**Changes made:**

1. **`src/lib/types/store.ts`** -- Added `OmnibarMode` type (`'file' | 'content'`) and `omnibarMode` field to `UiState`.

2. **`src/lib/stores/ui.ts`** -- Added `openOmnibar(mode)`, `closeOmnibar()`, and `setOmnibarMode(mode)` methods. Updated `toggleOmnibar()` to default to `'file'` mode when opening.

3. **`src/lib/features/omnibar/omnibar-store.ts`** -- Full rewrite to support dual-mode search:
   - Discriminated union types: `OmnibarFileResult` and `OmnibarContentResult`.
   - Separate `executeFileSearch` and `executeContentSearch` functions calling `fuzzyFind` and `searchContent` IPC respectively.
   - Mode-aware debouncing: 80 ms for file, 150 ms for content (per performance budget).
   - `setMode(mode)` re-runs current query in new mode.
   - `initialize(mode)` accepts initial mode.

4. **`src/lib/features/omnibar/Omnibar.svelte`** -- Added mode tab bar (Files / Content), Tab key to switch modes, `initialMode` prop, and mode-dependent placeholder text and empty messages. Updated onselect signature to include optional `lineNumber`.

5. **`src/lib/features/omnibar/OmnibarItem.svelte`** -- Handles both result types: file results show fuzzy-match highlighting on name, content results show file:line header and monospace preview with column-range highlighting.

6. **`src/lib/features/omnibar/index.ts`** -- Updated barrel exports for new types.

7. **`src/routes/+page.svelte`** -- Registered `Ctrl+Shift+F` shortcut to open omnibar in content mode. Updated `handleOmnibarSelect` to accept `lineNumber`. Passes `initialMode` from `uiState.omnibarMode` to Omnibar component.
