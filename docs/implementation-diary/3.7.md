# Implementation Diary: Task 3.7 -- Bidirectional Linking UI + [[ Autocomplete + Backlinks

## 2026-03-12T00:00 -- Initial Plan

**Goal:** Implement bidirectional linking UI with `[[` autocomplete and a backlinks panel that updates when link syntax is added or removed.

**Plan (6 steps):**

1. Create `src/lib/features/links/links-store.ts` -- reactive store that fetches backlinks for the active note via IPC, with a refresh method triggered on content changes.
2. Add wikilink extraction utilities to `src/lib/utils/markdown.ts` -- `extractWikilinks()` to parse `[[target]]` syntax from editor content for local link tracking.
3. Create `src/lib/features/links/WikilinkAutocomplete.svelte` -- dropdown triggered by `[[` in the textarea, fuzzy-searches workspace notes via `fuzzy_find` IPC, and inserts the selected link.
4. Create `src/lib/features/links/BacklinksPanel.svelte` -- collapsible panel showing notes that link to the active file, with source path, context snippet, and click-to-navigate.
5. Integrate both components into `NoteEditor.svelte` -- wire `[[` detection to autocomplete, render BacklinksPanel below editor, refresh backlinks on content save.
6. Run all checks: `npm run check`, `npm run lint`, `npm run build`, `cargo test`.
