4. Run all required checks (`npm run check`, `npm run lint`, `npm run build`, `cargo test`).

## 2026-03-12T00:01 -- Implementation Complete

**Files created/modified:**

1. **`src/lib/features/git/gutter.ts`** (NEW) -- Core gutter logic:
   - `hunksToDecorations()`: Converts `GitHunk[]` to Monaco `IModelDeltaDecoration[]` with glyph-margin classes.
   - `GutterController` class: Manages decoration lifecycle for a single editor instance.
     - `setFilePath()`: Triggers immediate hunk fetch on file switch.
     - `scheduleRefresh()`: Debounced (300ms) refresh for edit-triggered updates.
     - `applyHunks()`: Direct decoration application for backend push events.
     - `refreshImmediate()`: Fetches hunks via IPC, updates git store, applies decorations.
     - `dispose()`: Cleanup (cancel timers, clear decorations).
   - Stale response guard: ignores IPC results after file switch or disposal.

2. **`src/lib/features/editor/CodeEditor.svelte`** (MODIFIED) -- Integrated gutter controller:
   - Creates `GutterController` on mount, sets file path.
   - Subscribes to `vigil://git-hunks` backend events via `onGitHunks()`.
   - Calls `scheduleRefresh()` on every content edit (via `onDidChangeModelContent`).
   - Updates gutter on file switch (via `$effect` that tracks `filePath`).
   - Properly disposes controller and event listener on destroy.

3. **`src/lib/features/git/index.ts`** (MODIFIED) -- Re-exports `GutterController`, `hunksToDecorations`, and CSS class constants.

4. **`src/app.css`** (MODIFIED) -- Added `.vigil-gutter-added`, `.vigil-gutter-modified`, `.vigil-gutter-deleted` CSS classes for glyph-margin styling.
