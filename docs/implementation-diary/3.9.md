# Implementation Diary: Task 3.9 -- Interactive Graph View MVP

## 2026-03-12T00:00 -- Initial Plan

**Goal:** Implement interactive graph view (open/select/pan/zoom MVP).
Nodes represent notes, edges represent wikilinks. The graph renders in the sidebar graph panel.

**Plan (3-6 steps):**

1. **Add graph types:** Add `NoteNode`, `LinkEdge`, `NoteGraphResponse` to `src/lib/types/ipc.ts` and export from barrel. Add `getNoteGraph` IPC wrapper to `src/lib/ipc/links.ts`.
2. **Create graph store:** Build `src/lib/features/graph/graph-store.ts` with force-directed layout simulation, pan/zoom state, node selection, and data fetching logic.
3. **Rewrite GraphView.svelte:** Replace placeholder with interactive SVG canvas. Implement:
   - Force-directed node layout (simple spring-charge simulation)
   - Pan via mouse drag on canvas
   - Zoom via mouse wheel
   - Node selection on click (highlight node + edges)
   - Double-click node to open file (dispatch to editor store)
4. **Wire into sidebar:** Ensure GraphView is properly rendered in the sidebar graph panel with node/edge count footer.
5. **Run checks:** `npm run check`, `npm run lint`, `npm run build`, `cargo test`.
6. **Fix issues and finalize diary.**

**Key decisions:**
- Use pure SVG with `<g>` transform for pan/zoom (no external library dependency).
- Force layout runs client-side in a requestAnimationFrame loop.
- Since `get_note_graph` is an Epic 4 backend command (not yet implemented), the frontend will use mock data as fallback when the IPC call fails, matching the pattern used elsewhere.

## 2026-03-12T00:02 -- Step 2: Graph Store

Created `src/lib/features/graph/graph-store.ts` with:
- `GraphState` interface: nodes, edges, selectedNodeId, hoveredNodeId, transform, isSimulating, isLoading, error.
- `LayoutNode` type with position + velocity for force simulation.
- Force-directed simulation with repulsion, spring attraction, center gravity, damping.
- Pan/zoom methods operating on `ViewTransform` (offsetX, offsetY, scale).
- Node selection, hover, pin, move operations.
- Mock data fallback when backend unavailable.
- Derived stores: `selectedEdges`, `selectedNeighbors` for highlighting.

## 2026-03-12T00:03 -- Step 3: GraphView.svelte Rewrite

Replaced placeholder SVG with fully interactive component:
- SVG canvas with `<g>` transform for pan/zoom.
- Mouse drag on background = pan; mouse drag on node = reposition.
- Mouse wheel = zoom centered on cursor.
- Click node = select (highlights connected edges/neighbors, dims others).
- Double-click node = open file via editorStore.openFile().
- Keyboard: Enter/Space on focused node = select, Escape = deselect.
- Toolbar: Reset view, Zoom in, Zoom out buttons.
- Footer: node/edge count + selected node name.
- Tooltip on hover showing file path and tags.
- Pinned node indicator (yellow dot).

**What remains / risks:**

- `cargo test` could not be run (cargo not available in environment). No Rust files were modified, so this is low risk.

- SVG a11y warnings for the interactive canvas are expected and acceptable -- the SVG acts as an interactive application widget, not a static image.
