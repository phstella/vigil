# Task 0.3: Configure SvelteKit for Static SPA Output

## Goal

Disable SSR and configure SvelteKit to produce a static single-page app
compatible with Tauri's webview (no Node server at runtime).

## Commands

```bash
npm install -D @sveltejs/adapter-static
```

## Files to Modify

### `svelte.config.js`

Replace the default adapter with adapter-static:

```js
import adapter from '@sveltejs/adapter-static';
import { vitePreprocess } from '@sveltejs/vite-plugin-svelte';

/** @type {import('@sveltejs/kit').Config} */
const config = {
  preprocess: vitePreprocess(),
  kit: {
    adapter: adapter({
      pages: 'build',
      assets: 'build',
      fallback: 'index.html',
      precompress: false,
      strict: true
    })
  }
};

export default config;
```

### `src/routes/+layout.ts` (create new file)

```ts
export const ssr = false;
export const prerender = false;
```

This ensures:
- `ssr = false`: No server-side rendering — `window.__TAURI__` is always available
- `prerender = false`: Pages are not statically prerendered at build time
- `fallback: 'index.html'`: All routes resolve to the SPA shell

## Acceptance Criteria

- [ ] `npm run build` produces a `build/` directory containing `index.html`
- [ ] The `build/` directory contains bundled JS and CSS (no server files)
- [ ] `npx tauri build` packages the SPA into a native binary
- [ ] The built binary launches and renders the app correctly
