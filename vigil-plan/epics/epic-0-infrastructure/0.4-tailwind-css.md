# Task 0.4: Install & Configure Tailwind CSS v4

## Goal

Add Tailwind CSS v4 via the Vite plugin and define the Vigil glassmorphism
design tokens as a custom `@theme`.

## Commands

```bash
npm install tailwindcss @tailwindcss/vite
```

## Files to Modify

### `vite.config.ts`

Add the Tailwind Vite plugin (must come before sveltekit):

```ts
import { sveltekit } from '@sveltejs/kit/vite';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [tailwindcss(), sveltekit()]
});
```

### `src/app.css` (create new file)

```css
@import "tailwindcss";

@theme {
  --color-vigil-bg:         #0d0f12;
  --color-vigil-surface:    rgba(20, 24, 33, 0.75);
  --color-vigil-border:     rgba(255, 255, 255, 0.06);
  --color-vigil-teal:       #2dd4bf;
  --color-vigil-orange:     #fb923c;
  --color-vigil-green:      #4ade80;
  --color-vigil-text:       #e2e8f0;
  --color-vigil-text-muted: #64748b;
  --font-family-sans:       'Inter', system-ui, sans-serif;
  --font-family-mono:       'JetBrains Mono', 'Fira Code', monospace;
}
/* Sans-serif for Markdown prose, monospace for code files + code blocks */
```

### `src/routes/+layout.svelte`

Import the CSS and set up the root layout:

```svelte
<script>
  import '../app.css';
  let { children } = $props();
</script>

{@render children()}
```

## Notes

- Tailwind v4 does **not** use a `tailwind.config.js` file. All
  customization is done in CSS via `@theme` directives.
- No PostCSS config is needed — the `@tailwindcss/vite` plugin handles
  everything.

## Acceptance Criteria

- [ ] Tailwind utility classes work (e.g., `class="text-red-500"` renders red text)
- [ ] Custom tokens work (e.g., `bg-vigil-bg` applies `#0d0f12`)
- [ ] `backdrop-blur-xl` produces a visible blur effect on semi-transparent backgrounds
- [ ] `font-sans` uses Inter for prose, `font-mono` uses JetBrains Mono for code
- [ ] Both font families are available via `--font-family-sans` and `--font-family-mono`
- [ ] No `tailwind.config.js` or `postcss.config.js` files exist
