# Task 0.5: Add Rust Dependencies to Cargo.toml

## Goal

Declare all Rust crate dependencies required for the MVP backend so they
resolve and compile before any feature work begins.

## Files to Modify

### `src-tauri/Cargo.toml`

Add under `[dependencies]`:

```toml
serde = { version = "1", features = ["derive"] }
serde_json = "1"
git2 = "0.20"
notify = "8"
notify-debouncer-full = "0.5"
tantivy = "0.25"
rayon = "1.11"
tokio = { version = "1", features = ["full"] }
walkdir = "2"
fuzzy-matcher = "0.3"
trash = "5"
```

## Commands

Verify all crates resolve and compile:

```bash
cd src-tauri
cargo check
```

This will download and compile all dependencies. First run may take 2-5
minutes depending on the machine.

## Dependency Purposes

| Crate | Purpose |
|---|---|
| `serde` / `serde_json` | Serialize/deserialize all IPC payloads between Rust ↔ Svelte |
| `git2` | Git repository operations: status, diff, blame |
| `notify` | Cross-platform filesystem event watching |
| `notify-debouncer-full` | Debounce rapid filesystem events (300ms window) |
| `tantivy` | Full-text search engine (Lucene-like, pure Rust) |
| `rayon` | Data parallelism for file indexing operations |
| `tokio` | Async runtime for background task scheduling |
| `walkdir` | Recursive directory traversal (faster than std) |
| `fuzzy-matcher` | Skim-based fuzzy string matching for Ctrl+P |
| `trash` | Move files to OS trash instead of permanent deletion |

## Acceptance Criteria

- [ ] `cargo check` in `src-tauri/` completes without errors
- [ ] All 11 crates (plus transitive deps) resolve from crates.io
- [ ] `npx tauri dev` still launches successfully with the new dependencies
