# Task 0.8: Configure Tauri v2 Capabilities & Permissions

## Goal

Grant the frontend access to required Tauri APIs via the v2 capability/permission
system so that window controls, file dialogs, and IPC commands work without
runtime permission errors.

## Files to Modify

### `src-tauri/capabilities/default.json`

```json
{
  "$schema": "../gen/schemas/desktop-schema.json",
  "identifier": "default",
  "description": "Default permissions for Vigil",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "core:window:default",
    "core:window:allow-set-title",
    "core:window:allow-close",
    "core:window:allow-minimize",
    "core:window:allow-maximize",
    "core:window:allow-start-dragging",
    "core:event:default",
    "core:event:allow-listen",
    "core:event:allow-emit"
  ]
}
```

### `src-tauri/tauri.conf.json`

Verify the window configuration:

```json
{
  "app": {
    "windows": [
      {
        "title": "Vigil",
        "width": 1280,
        "height": 800,
        "minWidth": 800,
        "minHeight": 600,
        "decorations": true,
        "transparent": false
      }
    ]
  }
}
```

> Note: `decorations: false` and `transparent: true` will be set in Task 2.1
> when the custom title bar is built. For now, keep native decorations to
> avoid a non-functional window during early development.

## Tauri v2 Permission Model Notes

- Tauri v2 uses a **capability-based** permission system (replacing v1's
  allowlist). Each capability grants specific API access to specific windows.
- Custom Tauri commands (from Epic 1) are automatically allowed when
  registered via `generate_handler![]` — no additional permission config
  is needed for `invoke()` calls to custom commands.
- Additional permissions (e.g., `dialog:allow-open`, `fs:default`) will
  be added as features require them.

## Acceptance Criteria

- [ ] `npx tauri dev` launches without any permission-denied errors
- [ ] Window minimize/maximize/close controls work from the webview
- [ ] `window.__TAURI__` is accessible in the browser console
- [ ] `listen()` and `emit()` work for custom events (test with a simple ping)
