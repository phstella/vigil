# Task 1.2: Workspace Open & Recent Files

## Goal

Allow opening a directory as a "vault/workspace" and persisting a list of
recently opened workspaces across app restarts.

## Files to Modify

### `src-tauri/src/commands/workspace.rs`

```rust
#[derive(Serialize)]
pub struct FileEntry {
    pub name: String,
    pub path: String,     // relative to workspace root
    pub is_dir: bool,
    pub extension: Option<String>,
    pub depth: usize,     // nesting depth for tree rendering
}
```

Implement:

```rust
#[tauri::command]
pub fn open_workspace(path: String) -> Result<Vec<FileEntry>, String>
```
- Use `walkdir` to recursively traverse the directory.
- Skip hidden directories (starting with `.`), `node_modules`, `target/`, and `__pycache__`.
- Return a flat list of `FileEntry` objects with `depth` for tree construction on the frontend.
- Sort: directories first at each level, then alphabetical.

```rust
#[tauri::command]
pub fn get_recent_workspaces() -> Result<Vec<String>, String>
```
- Read from `~/.config/vigil/recent.json` (Linux) or `%APPDATA%/vigil/recent.json` (Windows).
- Use `tauri::api::path::config_dir()` for cross-platform path resolution.
- Return empty vec if file doesn't exist yet.

```rust
#[tauri::command]
pub fn save_recent_workspace(path: String) -> Result<(), String>
```
- Load existing list, prepend the new path, deduplicate, cap at 10 entries.
- Write back to `recent.json`.

### `src-tauri/src/main.rs`

Register the new commands in `generate_handler![]`.

## Performance Target

Opening a workspace with 1000+ files must return the full tree in **< 200ms**.

Use `rayon` for parallel directory entry processing if single-threaded walkdir
proves too slow on large vaults.

## Acceptance Criteria

- [ ] `invoke('open_workspace', { path: '/home/user/notes' })` returns the full file tree
- [ ] Hidden directories and `node_modules` are excluded from results
- [ ] `invoke('save_recent_workspace', { path: '/home/user/notes' })` persists to disk
- [ ] `invoke('get_recent_workspaces')` returns the saved list across app restarts
- [ ] A workspace with 1000 files loads in < 200ms (measure with `performance.now()`)
