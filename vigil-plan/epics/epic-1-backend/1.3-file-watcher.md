# Task 1.3: Recursive File Watcher

## Goal

Watch the active workspace for file changes and emit real-time events to the
frontend so the UI stays in sync with external modifications.

## Files to Modify

### `src-tauri/src/indexer/file_watcher.rs`

Implement:

```rust
pub struct FileWatcher {
    // Handle to stop the watcher when workspace changes
    stop_tx: Option<std::sync::mpsc::Sender<()>>,
}

impl FileWatcher {
    pub fn new() -> Self;
    pub fn watch(&mut self, path: &str, app_handle: tauri::AppHandle) -> Result<(), String>;
    pub fn stop(&mut self);
}
```

**Behavior:**
- Use `notify-debouncer-full` with a 300ms debounce window to batch rapid events.
- Spawn a dedicated `std::thread` for the watcher (not on the Tauri main thread).
- On each debounced event, classify and emit Tauri events:

| Filesystem Event | Tauri Event Name | Payload |
|---|---|---|
| File created | `vigil://fs-created` | `{ path: String }` |
| File modified | `vigil://fs-modified` | `{ path: String }` |
| File deleted | `vigil://fs-deleted` | `{ path: String }` |
| File renamed | `vigil://fs-renamed` | `{ old_path: String, new_path: String }` |

- Ignore events for hidden files/directories, `node_modules`, `target/`, `.git/` internals.
- Emit events using `app_handle.emit("vigil://fs-modified", payload)`.

### Integration with workspace open

In `commands/workspace.rs`, after `open_workspace`:
- Start the file watcher on the workspace root.
- Stop any previously running watcher.

## Acceptance Criteria

- [ ] With `npx tauri dev` running and a workspace open, run `touch new-file.md` in the workspace — `vigil://fs-created` event fires
- [ ] Edit a file externally — `vigil://fs-modified` event fires
- [ ] Delete a file — `vigil://fs-deleted` event fires
- [ ] Rename a file — `vigil://fs-renamed` event fires with old and new paths
- [ ] Rapid saves (e.g., holding Ctrl+S) produce debounced events, not a flood
- [ ] Events are visible in the browser console via `listen('vigil://fs-modified', ...)`
- [ ] Switching workspaces stops the old watcher and starts a new one
