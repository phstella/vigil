# Task 1.4: Git Status & Diff on Background Thread

## Goal

Provide line-level git diff data for the editor gutter and repo-wide file
status, without blocking the UI thread.

## Files to Modify

### `src-tauri/src/git/differ.rs`

Define types:

```rust
#[derive(Serialize, Clone)]
pub enum DiffKind {
    Added,
    Removed,
    Modified,
}

#[derive(Serialize, Clone)]
pub struct DiffHunk {
    pub start_line: usize,
    pub end_line: usize,
    pub kind: DiffKind,
}

#[derive(Serialize, Clone)]
pub struct FileStatus {
    pub path: String,
    pub status: String, // "modified", "added", "deleted", "untracked"
}
```

Implement:

```rust
pub struct GitDiffer {
    repo: git2::Repository,
}

impl GitDiffer {
    pub fn open(workspace_path: &str) -> Result<Self, String>;
    pub fn file_diff(&self, file_path: &str) -> Result<Vec<DiffHunk>, String>;
    pub fn repo_status(&self) -> Result<Vec<FileStatus>, String>;
}
```

**`file_diff` logic:**
- Get the blob for `file_path` at HEAD.
- Diff against the working tree version.
- Walk diff hunks and map each to `DiffHunk { start_line, end_line, kind }`.
- Handle new files (all lines are `Added`) and deleted files (all `Removed`).

**`repo_status` logic:**
- Use `repo.statuses(None)` to get all changed files.
- Map git2 status flags to simplified status strings.

### `src-tauri/src/commands/git.rs`

Expose as Tauri commands:

```rust
#[tauri::command]
pub async fn git_file_diff(workspace: String, path: String) -> Result<Vec<DiffHunk>, String>
```
- Use `tokio::task::spawn_blocking` to run `GitDiffer::file_diff` off the main thread.

```rust
#[tauri::command]
pub async fn git_repo_status(workspace: String) -> Result<Vec<FileStatus>, String>
```
- Use `tokio::task::spawn_blocking` similarly.

### `src-tauri/src/main.rs`

Register the new commands.

## Performance Target

`git_file_diff` must respond in **< 50ms** for files under 10,000 lines.

## Acceptance Criteria

- [ ] In a git repo with uncommitted changes, `invoke('git_file_diff', { workspace: '...', path: 'README.md' })` returns correct hunk data
- [ ] Returned hunks match `git diff` output (same line ranges and change types)
- [ ] `invoke('git_repo_status', { workspace: '...' })` lists all modified/added/deleted/untracked files
- [ ] New (untracked) files report status `"untracked"`
- [ ] Commands don't block the UI — the window remains responsive during a diff of a large file
- [ ] Non-git directories return a clean error (not a crash)
