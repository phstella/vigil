# Task 1.5: Fuzzy File Finder (Rust)

## Goal

Power the Ctrl+P omnibar with a fast Rust-based fuzzy-matching engine that
searches file paths within the active workspace.

## Files to Modify

### `src-tauri/src/commands/search.rs`

Define types:

```rust
#[derive(Serialize)]
pub struct FuzzyResult {
    pub path: String,           // relative path from workspace root
    pub score: i64,             // match quality score (higher = better)
    pub matched_indices: Vec<usize>, // character positions that matched
}
```

Implement:

```rust
use fuzzy_matcher::skim::SkimMatcherV2;
use fuzzy_matcher::FuzzyMatcher;
use std::sync::{Arc, RwLock};

// Global file path cache, updated by file watcher
static FILE_PATHS: Lazy<Arc<RwLock<Vec<String>>>> = ...;

#[tauri::command]
pub fn fuzzy_find(query: String, limit: usize) -> Result<Vec<FuzzyResult>, String>
```

**Logic:**
1. Acquire a read lock on the file paths cache.
2. Score every path against the query using `SkimMatcherV2`.
3. Collect results with score > 0.
4. Sort descending by score.
5. Return the top `limit` results with matched character indices.

**Cache management:**
- `refresh_file_paths(workspace: &str)` — called on workspace open, populates
  the cache using `walkdir`.
- On `vigil://fs-created` event: add the new path.
- On `vigil://fs-deleted` event: remove the path.
- On `vigil://fs-renamed` event: remove old, add new.

### Integration

- Call `refresh_file_paths` from `open_workspace` in `commands/workspace.rs`.
- Hook file-watcher events to cache update functions.

## Performance Target

Query must complete in **< 5ms** for a workspace with **50,000 files**.

The Skim algorithm is O(n) per query where n = number of paths, with a small
constant factor. For 50k paths this should be well under the target.

## Acceptance Criteria

- [ ] `invoke('fuzzy_find', { query: 'readme', limit: 10 })` returns ranked file paths
- [ ] Results include `matched_indices` for highlighting in the UI
- [ ] Partial/scattered matches work: `vigcnf` matches `vigil.config.ts`
- [ ] Results are ordered by match quality (exact prefix match > scattered match)
- [ ] Creating a new file and immediately searching for it returns the file
- [ ] Empty query returns an empty array (not all files)
- [ ] Query time < 5ms on a workspace with 50k file paths (log timing in dev)
