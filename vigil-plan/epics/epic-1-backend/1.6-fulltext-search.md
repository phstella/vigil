# Task 1.6: Full-Text Search Index

## Goal

Enable workspace-wide content search via a Tantivy full-text search index
with incremental updates on file changes.

## Files to Modify

### `src-tauri/src/indexer/search_index.rs`

Define types:

```rust
#[derive(Serialize)]
pub struct SearchHit {
    pub path: String,       // relative path from workspace root
    pub snippet: String,    // matching text excerpt with context
    pub score: f32,         // relevance score
    pub line_number: usize, // approximate line of the match
}
```

Implement:

```rust
pub struct SearchIndex {
    index: tantivy::Index,
    reader: tantivy::IndexReader,
    writer: tantivy::IndexWriter,
    schema: tantivy::schema::Schema,
}

impl SearchIndex {
    pub fn open_or_create(workspace: &str) -> Result<Self, String>;
    pub fn build_full(workspace: &str) -> Result<(), String>;
    pub fn update_file(&mut self, path: &str, content: &str) -> Result<(), String>;
    pub fn remove_file(&mut self, path: &str) -> Result<(), String>;
    pub fn search(&self, query: &str, limit: usize) -> Result<Vec<SearchHit>, String>;
}
```

**Schema fields:**
- `path` (TEXT, STORED) â€” relative file path
- `content` (TEXT) â€” full file content for searching
- `modified_at` (DATE, STORED) â€” last modified timestamp

**Index location:** `~/.cache/vigil/<workspace-path-hash>/index/`

Hash the workspace path to create a unique cache directory per workspace.

**File types to index:**
`.md`, `.txt`, `.rs`, `.ts`, `.tsx`, `.js`, `.jsx`, `.svelte`, `.css`,
`.json`, `.toml`, `.yaml`, `.yml`, `.py`, `.go`, `.html`

Skip binary files and files larger than 1MB.

**`build_full` logic:**
1. Clear existing index.
2. Walk workspace with `walkdir`.
3. Use `rayon` to parallelize file reading and tokenization.
4. Add all documents to the index writer.
5. Commit.

**`update_file` logic:**
1. Delete the existing document for `path`.
2. Re-add with new content.
3. Commit.

**Snippet generation:**
Use Tantivy's snippet generator to produce ~100-character excerpts with the
query terms highlighted (prefix/suffix markers for frontend rendering).

### `src-tauri/src/commands/search.rs`

Add Tauri command:

```rust
#[tauri::command]
pub async fn search_content(
    workspace: String,
    query: String,
    limit: usize
) -> Result<Vec<SearchHit>, String>
```

### Integration

- Call `build_full` on workspace open (run on background thread).
- On `vigil://fs-modified` events: call `update_file`.
- On `vigil://fs-deleted` events: call `remove_file`.
- On `vigil://fs-created` events: call `update_file` with new content.

## Performance Targets

- Full index build: < 5 seconds for 10,000 files
- Incremental update: < 50ms per file
- Search query: < 100ms for any query

## Acceptance Criteria

- [ ] After indexing, `invoke('search_content', { workspace: '...', query: 'TODO refactor', limit: 20 })` returns matching files
- [ ] Results include snippets with query terms identifiable for highlighting
- [ ] Results include approximate line numbers for navigation
- [ ] Editing a file and searching again returns updated results (incremental update)
- [ ] Binary files and files > 1MB are skipped without errors
- [ ] Index persists across app restarts (no rebuild needed if files haven't changed)
- [ ] Search completes in < 100ms
