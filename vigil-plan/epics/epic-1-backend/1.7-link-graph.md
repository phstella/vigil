# Task 1.7: Bidirectional Link Graph

## Goal

Parse `[[wikilinks]]` from Markdown files and maintain an in-memory
bidirectional link graph for instant backlink resolution.

## Files to Modify

### `src-tauri/src/indexer/link_graph.rs`

Define types:

```rust
use std::collections::{HashMap, HashSet};

pub struct LinkGraph {
    /// Forward links: file → set of files it links to
    forward: HashMap<String, HashSet<String>>,
    /// Reverse links (backlinks): file → set of files that link to it
    reverse: HashMap<String, HashSet<String>>,
}
```

Implement:

```rust
impl LinkGraph {
    pub fn new() -> Self;

    /// Extract all [[wikilink]] targets from content
    pub fn parse_links(content: &str) -> Vec<String>;

    /// Rebuild the entire graph by scanning all .md files in the workspace
    pub fn rebuild(&mut self, workspace: &str) -> Result<(), String>;

    /// Update links for a single file (on save or fs-modified event)
    pub fn update_file(&mut self, file_path: &str, content: &str);

    /// Remove a file from the graph (on delete)
    pub fn remove_file(&mut self, file_path: &str);

    /// Get all files that link TO the given file
    pub fn get_backlinks(&self, file_path: &str) -> Vec<String>;

    /// Get all files that the given file links TO
    pub fn get_forward_links(&self, file_path: &str) -> Vec<String>;
}
```

**`parse_links` regex:**
```rust
// Matches [[target]] and [[target|display text]]
// Captures the target portion only
let re = Regex::new(r"\[\[([^\]|]+)(?:\|[^\]]+)?\]\]").unwrap();
```

**Link resolution rules:**
- `[[note-name]]` resolves to `note-name.md` in the workspace
- `[[path/to/note]]` resolves to `path/to/note.md`
- Case-insensitive matching (normalize to lowercase for graph keys)
- Links to nonexistent files are still tracked (for "dangling link" detection)

**`update_file` logic:**
1. Remove all existing forward links from the file.
2. Re-parse the file content for new links.
3. Add new forward links.
4. Rebuild reverse map entries for affected files.

### `src-tauri/src/commands/search.rs`

Add Tauri commands:

```rust
#[tauri::command]
pub fn get_backlinks(workspace: String, path: String) -> Result<Vec<String>, String>

#[tauri::command]
pub fn get_forward_links(workspace: String, path: String) -> Result<Vec<String>, String>
```

### Integration

- Call `rebuild` on workspace open.
- On `vigil://fs-modified` events for `.md` files: call `update_file`.
- On `vigil://fs-deleted` events: call `remove_file`.
- On `vigil://fs-created` events for `.md` files: call `update_file`.

## Acceptance Criteria

- [ ] Given `note-a.md` containing `[[note-b]]`, calling `invoke('get_backlinks', { workspace: '...', path: 'note-b.md' })` returns `["note-a.md"]`
- [ ] `[[note-b|display text]]` syntax is parsed correctly (target = `note-b`)
- [ ] Forward links: `invoke('get_forward_links', { workspace: '...', path: 'note-a.md' })` returns `["note-b.md"]`
- [ ] Editing a file to add/remove a wikilink updates the graph incrementally
- [ ] Links are case-insensitive: `[[Note-B]]` matches `note-b.md`
- [ ] Dangling links (to nonexistent files) are tracked without errors
- [ ] Deleting a file removes it from all link relationships
