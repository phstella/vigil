# Task 1.8: Tag Extraction & Index

## Goal

Parse tags from Markdown files (frontmatter YAML and inline `#tag` syntax) and
maintain an in-memory index for tag browsing and status bar display.

## Files to Create

### `src-tauri/src/indexer/tag_index.rs`

Define types:

```rust
use std::collections::{HashMap, HashSet};

#[derive(Serialize, Clone)]
pub struct TagInfo {
    pub name: String,
    pub count: usize,         // number of files with this tag
    pub files: Vec<String>,   // file paths containing this tag
}

pub struct TagIndex {
    /// tag name → set of file paths
    tags: HashMap<String, HashSet<String>>,
    /// file path → set of tags in that file
    file_tags: HashMap<String, HashSet<String>>,
}
```

Implement:

```rust
impl TagIndex {
    pub fn new() -> Self;

    /// Extract tags from file content.
    /// Supports:
    ///   - YAML frontmatter: `tags: [foo, bar]` or `tags:\n  - foo\n  - bar`
    ///   - Inline hashtags: `#tag-name` (word-boundary delimited, not inside code blocks)
    pub fn parse_tags(content: &str) -> Vec<String>;

    /// Rebuild the entire index by scanning all .md files
    pub fn rebuild(&mut self, workspace: &str) -> Result<(), String>;

    /// Update tags for a single file (on save or fs-modified)
    pub fn update_file(&mut self, file_path: &str, content: &str);

    /// Remove a file from the index
    pub fn remove_file(&mut self, file_path: &str);

    /// Get all tags with their file counts
    pub fn get_all_tags(&self) -> Vec<TagInfo>;

    /// Get tags for a specific file
    pub fn get_file_tags(&self, file_path: &str) -> Vec<String>;

    /// Get all files with a specific tag
    pub fn get_files_by_tag(&self, tag: &str) -> Vec<String>;
}
```

**Tag parsing rules:**
- Frontmatter is delimited by `---` at the start of the file.
- Inline `#tag` must be preceded by whitespace or start of line.
- Tags inside code blocks (``` or `inline`) are ignored.
- Tags are normalized to lowercase, trimmed, with special chars stripped.
- Multi-word tags via `#multi-word-tag` (hyphenated).

### Integration

- Call `rebuild` on workspace open (alongside link graph and search index).
- On `vigil://fs-modified` events for `.md` files: call `update_file`.
- On `vigil://fs-deleted` events: call `remove_file`.

## Acceptance Criteria

- [ ] YAML frontmatter `tags: [foo, bar]` is parsed correctly
- [ ] Inline `#project-alpha` is extracted as tag `project-alpha`
- [ ] Tags inside code blocks are ignored
- [ ] `invoke('get_all_tags', { workspace: '...' })` returns all tags with counts
- [ ] `invoke('get_file_tags', { workspace: '...', path: 'note.md' })` returns tags for a specific file
- [ ] `invoke('get_files_by_tag', { workspace: '...', tag: 'project' })` returns all files tagged `project`
- [ ] Editing a file to add/remove a tag updates the index incrementally
- [ ] Tags are case-insensitive (`#Foo` and `#foo` are the same tag)
