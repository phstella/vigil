# Task 2.2: Multi-Panel Sidebar with Auto-Hide

## Goal

Build a collapsible, auto-hiding sidebar that hosts multiple panel views
(Explorer, Search, Graph, Tags) switchable via the Activity Bar (Task 2.8).
The Explorer panel renders the workspace file tree with collections and
file counts.

## Files to Create

### `src/lib/components/layout/Sidebar.svelte`

**Structure:**
```svelte
<aside
  class="sidebar"
  class:collapsed={!visible}
  onmouseenter={cancelAutoHide}
  onmouseleave={scheduleAutoHide}
>
  {#if activePanel === 'explorer'}
    <ExplorerPanel />
  {:else if activePanel === 'search'}
    <SearchPanel />
  {:else if activePanel === 'graph'}
    <GraphView />
  {:else if activePanel === 'tags'}
    <TagsPanel />
  {/if}
</aside>
```

**Auto-hide behavior:**
- When the editor has focus for > 3 seconds and the mouse is not over the
  sidebar, the sidebar slides to a collapsed state (width → 0).
- Moving the mouse to the left edge (within 8px) or pressing `Ctrl+B`
  reveals the sidebar.
- Auto-hide can be disabled in settings (`autoHideSidebar: boolean`).
- The collapse/reveal animation is 200ms `ease-out`.

**Styling:**
```
w-60
bg-vigil-surface/50
backdrop-blur-lg
border-r border-vigil-border
overflow-y-auto overflow-x-hidden
transition-all duration-200 ease-out
shrink-0
```

Collapsed state:
```
w-0 overflow-hidden opacity-0
```

### `src/lib/components/sidebar/ExplorerPanel.svelte`

**Structure:**
```svelte
<div class="explorer-panel">
  <header class="panel-header">
    <span class="text-vigil-teal text-xs font-bold tracking-wider">EXPLORER</span>
  </header>

  <!-- Note count -->
  <div class="note-count">
    <span>Notes</span>
    <span class="badge">{noteCount}</span>
  </div>

  <!-- Collections (directories as groups) -->
  <div class="collections">
    <div class="section-header text-vigil-teal text-xs font-semibold">
      Collections
    </div>
    {#each collections as collection}
      <button class="collection-item" onclick={() => toggleCollection(collection)}>
        <span class="chevron">{collection.expanded ? '▼' : '▶'}</span>
        <span class="name">{collection.name}</span>
        <span class="count text-vigil-text-muted">{collection.count}</span>
      </button>
      {#if collection.expanded}
        {#each collection.files as file}
          <FileTreeNode {file} depth={1} />
        {/each}
      {/if}
    {/each}
  </div>

  <!-- Loose files (not in any collection) -->
  {#each looseFiles as file}
    <FileTreeNode {file} depth={0} />
  {/each}
</div>
```

**Collections concept:**
- Top-level directories in the workspace are presented as "Collections."
- Each collection shows its name and a file count badge.
- Collections are expandable/collapsible.
- Files not in any subdirectory appear below as loose files.
- The active file is highlighted with `bg-vigil-teal/10 text-vigil-teal`.

### `src/lib/components/sidebar/SearchPanel.svelte`

**Structure:**
```svelte
<div class="search-panel">
  <header class="panel-header">
    <span class="text-vigil-teal text-xs font-bold tracking-wider">SEARCH</span>
  </header>
  <input
    class="search-input"
    bind:value={query}
    placeholder="Search in files..."
  />
  <div class="search-results">
    {#each results as result}
      <SearchResultItem {result} />
    {/each}
  </div>

```

An always-visible search panel (alternative to the `Ctrl+Shift+F` omnibar
overlay). Calls `searchContent()` with a 200ms debounce. Results show
file path + snippet.

### `src/lib/components/sidebar/TagsPanel.svelte`

**Structure:**
```svelte
<div class="tags-panel">
  <header class="panel-header">
    <span class="text-vigil-teal text-xs font-bold tracking-wider">TAGS</span>
  </header>
  <div class="tag-list">
    {#each tags as tag}
      <button
        class="tag-item"
        class:active={selectedTag === tag.name}
        onclick={() => filterByTag(tag.name)}
      >
        <span class="tag-name">#{tag.name}</span>
        <span class="tag-count">{tag.count}</span>
      </button>
    {/each}
  </div>
  {#if selectedTag}
    <div class="filtered-files">
      {#each filteredFiles as file}
        <FileTreeNode {file} depth={0} />
      {/each}
    </div>
  {/if}
</div>
```

Clicking a tag filters the file list to only files with that tag. Tags are
sorted by count (most used first). Data comes from the Rust tag index
(Task 1.8).

## File Tree Node (shared component)

### `src/lib/components/sidebar/FileTreeNode.svelte`

Used across all panels. Renders a single file entry with:
- File-type icon (inline SVG): `.md` → document, `.py` → code, `.rs` → gear, etc.
- File name, truncated with ellipsis on overflow.
- Active file: `bg-vigil-teal/10 text-vigil-teal`.
- Clicking opens the file in the editor.
- Indent per depth level: `pl-{depth * 4}`.

- [ ] Search panel shows full-text results with snippets
