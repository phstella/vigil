# Task 2.3: Resizable Split-Pane Workspace (Horizontal + Vertical)

## Goal

Allow side-by-side or stacked editing of two files with a draggable divider,
supporting single-pane, horizontal split, and vertical split modes.

## Files to Create

### `src/lib/components/layout/SplitPane.svelte`

**Structure:**
```svelte
<div class="split-container">
  <div class="pane left" style="width: {leftWidth}%">
    <slot name="left" />
  </div>
  {#if splitMode}
    <div
      class="divider"
      onmousedown={startResize}
      ondblclick={resetSplit}
    />
    <div class="pane right" style="width: {100 - leftWidth}%">
      <slot name="right" />
    </div>
  {/if}
</div>
```

**Behavior:**
- **Single-pane mode** (default): one editor fills the full space.
- **Horizontal split**: toggled via `Ctrl+\`. Two side-by-side editor panes.
- **Vertical split**: toggled via `Ctrl+Shift+\`. Two stacked editor panes.
- **Draggable divider**: 4px wide/tall, `cursor-col-resize` or `cursor-row-resize`.
  - Dragging adjusts the split ratio in real time.
  - Constrain range: 20% – 80% (neither pane can collapse below 20%).
  - Double-click the divider to reset to 50/50.
- **Persist** the split ratio and orientation to `localStorage`.
- `splitOrientation` state: `'none' | 'horizontal' | 'vertical'`.

## Acceptance Criteria

- [ ] Default state: single pane, editor fills full space
- [ ] `Ctrl+\` toggles horizontal split — two panes side by side
- [ ] `Ctrl+Shift+\` toggles vertical split — two panes stacked top/bottom
- [ ] Dragging the divider resizes panes fluidly at 60fps (no jank)
- [ ] Double-clicking the divider resets to 50/50 split
- [ ] Panes cannot be resized below 20%
- [ ] Split ratio and orientation persist across page reloads via `localStorage`
- [ ] Each pane can independently host an editor instance (WYSIWYG or Monaco)
- [ ] In single-pane mode, the divider and second pane are not rendered
