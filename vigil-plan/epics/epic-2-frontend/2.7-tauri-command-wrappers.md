# Task 2.7: Tauri Command Wrappers (TypeScript)

## Goal

Create typed, ergonomic wrappers around all Tauri `invoke()` and `listen()`
calls so the frontend never uses raw IPC directly.

## Files to Create

### `src/lib/tauri/commands.ts`

### `src/lib/tauri/events.ts`

Typed event listeners with automatic cleanup:

```ts
import { listen, type UnlistenFn } from '@tauri-apps/api/event';

interface FsEventPayload {
  path: string;
}

interface FsRenamePayload {
  old_path: string;
  new_path: string;
}

export async function onFsCreated(
  callback: (path: string) => void
): Promise<UnlistenFn> {
  return listen<FsEventPayload>('vigil://fs-created', (event) => {
    callback(event.payload.path);
  });
}

export async function onFsModified(
  callback: (path: string) => void
): Promise<UnlistenFn> {
  return listen<FsEventPayload>('vigil://fs-modified', (event) => {
    callback(event.payload.path);
  });
}

export async function onFsDeleted(
  callback: (path: string) => void
): Promise<UnlistenFn> {
  return listen<FsEventPayload>('vigil://fs-deleted', (event) => {
    callback(event.payload.path);
  });
}

export async function onFsRenamed(
  callback: (oldPath: string, newPath: string) => void
): Promise<UnlistenFn> {
  return listen<FsRenamePayload>('vigil://fs-renamed', (event) => {
    callback(event.payload.old_path, event.payload.new_path);
  });
}
```

## Design Decisions

- **No raw `invoke()` in components**: All IPC goes through these wrappers.
  This provides a single place to add error handling, logging, or caching.
- **Types mirror Rust structs**: Field names use `snake_case` to match Rust
  serialization. The frontend consumes them as-is.
- **Event listeners return `UnlistenFn`**: Callers must store and call the
  unlisten function on component destroy to prevent memory leaks.
- **Graph + Tags**: New command groups for `getFullLinkGraph`, `getAllTags`,
  `getFileTags`, and `getFilesByTag` support the Graph View and Tags Panel.

## Acceptance Criteria

- [ ] Frontend code never calls raw `invoke()` — all usage goes through `commands.ts`
- [ ] All functions have correct TypeScript types matching Rust return types
- [ ] TypeScript compilation passes in strict mode
- [ ] Event listeners properly type their payloads
- [ ] `UnlistenFn` is returned and documented for cleanup
- [ ] Importing from `$lib/tauri/commands` works from any component
