# Task 3.1: Monaco Editor (Code Files)

## Goal

Embed Monaco Editor as the editing surface for **non-Markdown source files**
(`.py`, `.rs`, `.ts`, `.js`, `.json`, etc.) with a custom Vigil dark theme,
bidirectional binding to the editor store, and zero-lag input.

> **Note:** Markdown files use the WYSIWYG editor (Task 3.2) instead. An
> `EditorRouter` component (see Task 2.5) switches between them based on
> file extension.

## Commands

```bash
npm install monaco-editor
```

## Files to Create

### `src/lib/components/editor/Editor.svelte`

**Initialization:**
```ts
import * as monaco from 'monaco-editor';

let editorContainer: HTMLDivElement;
let editorInstance: monaco.editor.IStandaloneCodeEditor;

// Map of file path → model (preserves undo history per file)
const modelCache = new Map<string, monaco.editor.ITextModel>();

onMount(() => {
  registerVigilTheme();
  editorInstance = monaco.editor.create(editorContainer, {
    theme: 'vigil-dark',
    fontFamily: "'JetBrains Mono', 'Fira Code', monospace",
    fontSize: settingsState.fontSize,
    tabSize: settingsState.tabSize,
    wordWrap: settingsState.wordWrap ? 'on' : 'off',
    minimap: { enabled: false },
    scrollBeyondLastLine: false,
    renderLineHighlight: 'gutter',
    cursorBlinking: 'smooth',
    cursorSmoothCaretAnimation: 'on',
    smoothScrolling: true,
    padding: { top: 16 },
    automaticLayout: true, // auto-resize with container
  });

  // Cursor position reporting
  editorInstance.onDidChangeCursorPosition((e) => {
    editorState.cursorLine = e.position.lineNumber;
    editorState.cursorCol = e.position.column;
  });

  // Content change tracking
  editorInstance.onDidChangeModelContent(() => {
    editorState.isDirty = true;
    editorState.content = editorInstance.getValue();
  });
});
```

**Custom theme — `registerVigilTheme()`:**
```ts
monaco.editor.defineTheme('vigil-dark', {
  base: 'vs-dark',
  inherit: true,
  rules: [
    { token: 'keyword', foreground: '2dd4bf' },       // teal
    { token: 'string', foreground: '4ade80' },         // green
    { token: 'comment', foreground: '64748b' },        // muted
    { token: 'number', foreground: 'fb923c' },         // orange
    { token: 'type', foreground: '2dd4bf' },           // teal
    // Markdown-specific tokens
    { token: 'markup.heading', foreground: '2dd4bf', fontStyle: 'bold' },
    { token: 'markup.bold', fontStyle: 'bold' },
    { token: 'markup.italic', fontStyle: 'italic' },
    { token: 'string.link', foreground: 'fb923c' },
  ],
  colors: {
    'editor.background': '#00000000',                  // transparent
    'editor.foreground': '#e2e8f0',
    'editor.lineHighlightBackground': '#ffffff08',
    'editor.selectionBackground': '#2dd4bf30',
    'editorCursor.foreground': '#2dd4bf',
    'editorLineNumber.foreground': '#64748b',
    'editorLineNumber.activeForeground': '#e2e8f0',
  },
});
```

**File switching:**
```ts
// React to activeFile changes
$effect(() => {
  const file = editorState.activeFile;
  if (!file) return;

  // Check model cache first (preserves undo history)
  let model = modelCache.get(file);
  if (!model) {
    const lang = deriveMonacoLanguage(file);
    model = monaco.editor.createModel(editorState.content, lang);
    modelCache.set(file, model);
  }
  editorInstance.setModel(model);
});
```

**Save handler:**
```ts
// Ctrl+S is handled by the keybinding registry (Task 3.7)
// which calls this function:
export async function saveCurrentFile() {
  if (!editorState.activeFile || !editorState.isDirty) return;
  await writeFile(editorState.activeFile, editorInstance.getValue());
  editorState.isDirty = false;
}
```

**Cleanup:**
```ts
onDestroy(() => {
  editorInstance?.dispose();
  modelCache.forEach((model) => model.dispose());
  modelCache.clear();
});
```

## Monaco Bundle Optimization

Configure Vite to only include needed languages. In `vite.config.ts`, use
Monaco's ESM build to enable tree-shaking. Only register languages for:
Markdown, TypeScript, JavaScript, Rust, JSON, CSS, HTML, YAML, TOML, Python.

## Acceptance Criteria

- [ ] Opening a `.py` or `.rs` file renders syntax-highlighted code
- [ ] Opening a `.md` file routes to WysiwygEditor instead (not Monaco)
- [ ] Typing has zero perceptible input lag (< 16ms frame time)
- [ ] Custom Vigil theme applies: transparent background, teal cursor, teal keywords
- [ ] `Ctrl+S` saves the file to disk and clears the dirty flag
- [ ] Switching files preserves undo history (model cache)
- [ ] Cursor position updates in the status bar on every move
- [ ] Editor auto-resizes when the window or split pane is resized
- [ ] Disposing the component cleans up all Monaco instances (no memory leaks)
- [ ] Editor background is transparent (parent glassmorphism shows through)
