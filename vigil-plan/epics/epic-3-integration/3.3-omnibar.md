# Task 3.3: Ctrl+P Omnibar — Fuzzy File Finder

## Goal

Build a floating command palette for instant file navigation, powered by the
Rust fuzzy-matching backend from Task 1.5.

## Files to Create

### `src/lib/components/omnibar/Omnibar.svelte`

**Structure:**
```svelte
{#if visible}
  <div class="omnibar-backdrop" onclick={close}>
    <div class="omnibar-card" onclick|stopPropagation>
      <input
        class="omnibar-input"
        bind:value={query}
        oninput={onQueryChange}
        onkeydown={onKeyDown}
        placeholder="Search files..."
        autofocus
      />
      <div class="omnibar-results">
        {#each results as result, i}
          <OmnibarItem
            {result}
            active={i === selectedIndex}
            onclick={() => openResult(result)}
          />
        {/each}
        {#if results.length === 0 && query.length > 0}
          <div class="no-results">No files found</div>
        {/if}
      </div>
    </div>
  </div>
{/if}
```

**Behavior:**
- **Open**: `Ctrl+P` (registered in keybinding registry).
- **Close**: `Escape` key or clicking the backdrop.
- **Search**: On every keystroke (debounced 30ms), call `fuzzyFind(query, 12)`.
- **Navigate**: `ArrowUp` / `ArrowDown` to move selection. `Enter` to open.
- **Open result**: Set `editorState.activeFile`, call `readFile()`, close omnibar.
- **Focus**: Input is auto-focused when the omnibar opens.

### `src/lib/components/omnibar/OmnibarItem.svelte`

**Structure:**
```svelte
<div class="omnibar-item" class:active>
  <!-- File type badge (colored pill) -->
  <span class="type-badge" style="background: {badgeColor}">
    {fileExtLabel}
  </span>

  <div class="item-details">
    <!-- File name with match highlighting -->
    <span class="file-name">
      {#each nameChars as char, i}
        <span class:highlight={result.matched_indices.includes(i)}>
          {char}
        </span>
      {/each}
    </span>

    <!-- Directory path + category label -->
    <span class="file-meta text-vigil-text-muted text-xs">
      /{result.directory} ({result.category})
    </span>
  </div>

  <!-- Active file indicator -->
  {#if result.isOpen}
    <span class="check-mark text-vigil-teal">✓</span>
  {/if}
</div>
```

**File type badges:**
- `MD` → blue-teal badge
- `PY` → yellow badge
- `RS` → orange badge
- `TS`/`JS` → blue badge
- Default → gray badge

**Category labels:**
Derived from the parent directory or file type: `(Note)`, `(Script)`,
`(Config)`, `(Source)`. This gives context about what each file is.

**Match highlighting:**
- Characters at positions in `matched_indices` get the `highlight` class.
- Highlighted characters: `text-vigil-teal font-bold`.

**Open file checkmark:**
If the file is currently open in an editor pane, show a `✓` on the right.
