# Task 3.4: Ctrl+Shift+F — Workspace Content Search

## Goal

Add full-text search across all workspace files via the Tantivy index,
accessible through a dedicated keybinding or an extension of the omnibar.

## Implementation Approach

Extend the existing Omnibar (Task 3.3) with a "content search" mode, rather
than building a separate UI. This keeps the interaction pattern consistent.

## Files to Modify

### `src/lib/components/omnibar/Omnibar.svelte`

Add a `mode` state: `'files' | 'content'`.

**Mode switching:**
- `Ctrl+P` opens in `files` mode (fuzzy file search).
- `Ctrl+Shift+F` opens in `content` mode (full-text search).
- Visual indicator: the input placeholder changes:
  - Files mode: `"Search files..."`
  - Content mode: `"Search in files..."`
- A small mode badge/pill next to the input: `[Files]` or `[Content]`.

**Content search behavior:**
- On keystroke (debounced 150ms — longer than file search since it's heavier),
  call `searchContent(workspace, query, 20)`.
- Display results as:
  ```
  [file-icon] path/to/file.md
    ...matching snippet with query terms highlighted...
  ```
- Clicking a result:
  1. Opens the file.
  2. Scrolls to the matching line (`result.line_number`).
  3. Highlights the search term in the editor.

### `src/lib/components/omnibar/OmnibarItem.svelte`

Extend to handle both result types:

```svelte
{#if result.type === 'file'}
  <!-- fuzzy match display (existing) -->
{:else if result.type === 'content'}
  <div class="content-result">
    <div class="file-path text-vigil-text-muted text-xs">{result.path}</div>
    <div class="snippet text-sm">
      {@html highlightSnippet(result.snippet, query)}
    </div>
  </div>
{/if}
```

**Snippet highlighting:**
Wrap query terms in `<mark class="bg-vigil-teal/30 text-vigil-teal">` tags.
Sanitize the snippet first to prevent XSS.

## Acceptance Criteria

- [ ] `Ctrl+Shift+F` opens the omnibar in content search mode
- [ ] Typing a query searches across all indexed workspace files
- [ ] Results show file path + matching snippet with highlighted terms
- [ ] Clicking a result opens the file and scrolls to the matching line
- [ ] Mode indicator shows whether you're in file or content search
- [ ] Content search debounces at 150ms (not on every keystroke)
- [ ] Results update correctly as the query changes
- [ ] Searching for a term that exists in multiple files returns all matches
- [ ] No XSS vulnerability from snippet rendering (HTML is sanitized)
