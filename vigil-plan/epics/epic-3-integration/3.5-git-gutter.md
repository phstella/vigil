# Task 3.5: Git Gutter Markers

## Goal

Show added, modified, and deleted line indicators in the editor gutter using
data from the Rust git diff backend (Task 1.4).

## Files to Create

### `src/lib/components/editor/GitGutter.svelte`

> Note: This is not a standalone visual component in the traditional sense.
> It's a module that manages Monaco editor decorations. The name follows
> the convention of the `editor/` directory.

**Implementation via Monaco Decorations API:**

```ts
import type { DiffHunk } from '$lib/tauri/commands';

const decorationIds: string[] = [];

export function applyGitDecorations(
  editor: monaco.editor.IStandaloneCodeEditor,
  hunks: DiffHunk[]
) {
  // Clear previous decorations
  editor.removeDecorations(decorationIds);

  const newDecorations: monaco.editor.IModelDeltaDecoration[] = hunks.map((hunk) => ({
    range: new monaco.Range(hunk.start_line, 1, hunk.end_line, 1),
    options: {
      isWholeLine: true,
      linesDecorationsClassName: getGutterClass(hunk.kind),
    },
  }));

  decorationIds = editor.deltaDecorations([], newDecorations);
}

function getGutterClass(kind: string): string {
  switch (kind) {
    case 'Added':    return 'git-gutter-added';
    case 'Modified': return 'git-gutter-modified';
    case 'Removed':  return 'git-gutter-deleted';
    default:         return '';
  }
}
```

## Integration with Editor (Task 3.1)

In `Editor.svelte`, after loading a file:

```ts
// Fetch and apply git diff decorations
$effect(() => {
  const file = editorState.activeFile;
  const workspace = filesState.workspacePath;
  if (!file || !workspace) return;

  gitFileDiff(workspace, file).then((hunks) => {
    applyGitDecorations(editorInstance, hunks);
    // Cache in git store for status bar use
    gitState.fileDiffs.set(file, hunks);
  }).catch(() => {
    // Not a git repo or file is untracked — clear decorations
    applyGitDecorations(editorInstance, []);
  });
});
```

**Re-fetch triggers:**
- File is opened (initial load).
- File is saved (`Ctrl+S` — diff may change after save).
- `vigil://fs-modified` event for the active file.

## Acceptance Criteria

- [ ] After opening a modified file in a git repo, colored bars appear in the gutter
- [ ] Added lines show a green bar (3px, left edge of gutter)
- [ ] Modified lines show an orange bar
- [ ] Deleted lines show a red bar at the deletion point
- [ ] Markers match `git diff` output (same line ranges)
- [ ] Saving a file refreshes the gutter markers
- [ ] Opening a file in a non-git directory shows no gutter markers (no errors)
- [ ] Opening an untracked (new) file shows all lines as "Added"
- [ ] Switching files updates the gutter to reflect the new file's diff
