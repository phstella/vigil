# Task 3.7: Keyboard Shortcut Registry

## Goal

Centralize all keybindings in a single registry for consistent behavior,
discoverability, and future customization.

## Files to Create

### `src/lib/utils/keybindings.ts`

```ts
type KeyHandler = () => void | Promise<void>;

interface Keybinding {
  key: string;         // e.g., 'ctrl+s', 'ctrl+shift+f'
  action: string;      // human-readable name, e.g., 'Save file'
  handler: KeyHandler;
  when?: string;       // optional context (e.g., 'editorFocus') — for future use
}

class KeybindingRegistry {
  private bindings: Map<string, Keybinding> = new Map();

  register(key: string, action: string, handler: KeyHandler): void {
    this.bindings.set(normalizeKey(key), { key, action, handler });
  }

  unregister(key: string): void {
    this.bindings.delete(normalizeKey(key));
  }

  /** Attach the global keydown listener */
  attach(): void {
    document.addEventListener('keydown', this.handleKeyDown);
  }

  /** Detach the global listener */
  detach(): void {
    document.removeEventListener('keydown', this.handleKeyDown);
  }

  private handleKeyDown = (e: KeyboardEvent): void => {
    const key = eventToKeyString(e);
    const binding = this.bindings.get(key);
    if (binding) {
      e.preventDefault();
      e.stopPropagation();
      binding.handler();
    }
  };

  /** Get all registered bindings (for a future shortcuts panel) */
  getAll(): Keybinding[] {
    return Array.from(this.bindings.values());
  }
}

// Normalize: 'Ctrl+Shift+F' → 'ctrl+shift+f'

// Convert KeyboardEvent to normalized string
function eventToKeyString(e: KeyboardEvent): string { ... }

export const keybindings = new KeybindingRegistry();
```

## Default Bindings

Register in `src/routes/+page.svelte` on mount:

```ts
import { keybindings } from '$lib/utils/keybindings';

onMount(() => {
  keybindings.register('ctrl+s',       'Save file',           saveCurrentFile);
  keybindings.register('ctrl+p',       'Open file finder',    openOmnibar);
  keybindings.register('ctrl+shift+f', 'Search in workspace', openWorkspaceSearch);
  keybindings.register('ctrl+b',       'Toggle sidebar',      toggleSidebar);
  keybindings.register('ctrl+\\',      'Toggle split pane',   toggleSplitPane);
  keybindings.register('ctrl+n',       'New file',            createNewFile);
  keybindings.register('ctrl+w',       'Close active file',   closeActiveFile);
  keybindings.register('ctrl+tab',     'Cycle open files',    cycleOpenFiles);
  keybindings.register('ctrl+,',       'Open settings',       openSettings);

  keybindings.attach();

  return () => keybindings.detach();
});
```

## Conflict Avoidance with Monaco

Monaco has its own keybinding system. To avoid conflicts:
- Keybindings that should be handled **globally** (like `Ctrl+P`, `Ctrl+B`)
  are registered in the global registry and `preventDefault()` before Monaco
  sees them.
- Keybindings that should work **only in the editor** (like `Ctrl+D` for
  multi-cursor) are left to Monaco's default bindings.
- `Ctrl+S` is handled globally (not by Monaco) to ensure it always triggers
  the save command.
