# Task 3.8: File Watcher → UI Sync

## Goal

Wire Tauri file-watcher events (from Task 1.3) to the frontend stores so the
UI reflects external file changes in real time.

## Files to Modify

### `src/routes/+page.svelte` (or a dedicated `src/lib/tauri/sync.ts` module)

Set up event listeners on workspace open:

```ts
import { onFsCreated, onFsModified, onFsDeleted, onFsRenamed } from '$lib/tauri/events';
import { filesState } from '$lib/stores/files';
import { editorState } from '$lib/stores/editor';
import { readFile } from '$lib/tauri/commands';

let unlisteners: Array<() => void> = [];

async function startFileSync() {
  unlisteners.push(
    await onFsCreated(handleCreated),
    await onFsModified(handleModified),
    await onFsDeleted(handleDeleted),
    await onFsRenamed(handleRenamed),
  );
}

function stopFileSync() {
  unlisteners.forEach((fn) => fn());
  unlisteners = [];
}
```

### Event Handlers

**`handleCreated(path: string)`:**
1. Add the new file entry to `filesState.fileTree` at the correct position.
2. Re-sort the tree (dirs first, alphabetical).

**`handleModified(path: string)`:**
1. If `path === editorState.activeFile`:
   - If `editorState.isDirty === false`: auto-reload the file content.
     ```ts
     const newContent = await readFile(path);
     editorState.content = newContent;
     // Update the Monaco model
     ```
   - If `editorState.isDirty === true`: show a conflict banner:
     ```
     ┌─────────────────────────────────────────────────┐
     │ ⚠ File changed on disk. [Reload] [Keep mine]    │
     └─────────────────────────────────────────────────┘
     ```
     - **Reload**: discard editor content, load from disk.
     - **Keep mine**: dismiss the banner, keep editor content (mark as dirty).
2. If `path !== editorState.activeFile`: invalidate any cached data for that
   file (e.g., git diff cache, model cache).

**`handleDeleted(path: string)`:**
1. Remove the file from `filesState.fileTree`.
2. If `path === editorState.activeFile`: show a "file deleted" banner:
   ```
   ┌─────────────────────────────────────────────────┐
   │ ⚠ This file was deleted externally. [Save as...] │
   └─────────────────────────────────────────────────┘
   ```
   - The editor content is preserved so the user can save it elsewhere.

**`handleRenamed(oldPath: string, newPath: string)`:**
1. Update the entry in `filesState.fileTree` (old path → new path).
2. If `oldPath === editorState.activeFile`:
   - Update `editorState.activeFile = newPath`.
   - Update the title bar display.

## Acceptance Criteria

- [ ] Creating a file externally (e.g., `touch new.md`) immediately adds it to the sidebar tree
- [ ] Deleting a file externally removes it from the sidebar tree
- [ ] Renaming a file externally updates its name in the tree and title bar
- [ ] Modifying the active file externally (while it's clean) auto-reloads the content
- [ ] Modifying the active file externally (while it's dirty) shows the conflict banner
- [ ] "Reload" button in the conflict banner loads the disk version
- [ ] "Keep mine" button dismisses the banner and preserves editor content
- [ ] Deleting the active file shows a "file deleted" banner with "Save as..." option
- [ ] All event listeners are properly cleaned up on workspace change or app close
- [ ] No race conditions between file-watcher events and user edits
