# Task 3.9: Graph View (Note Connection Visualization)

## Goal

Build an interactive force-directed graph visualization that renders note
connections from the bidirectional link graph (Task 1.7), displayed as a
sidebar panel accessible from the Activity Bar.

## Files to Create

### `src/lib/components/sidebar/GraphView.svelte`

**Architecture:**
```
GraphView.svelte
├── <canvas> element (2D Canvas API or WebGL)
├── Force simulation (requestAnimationFrame loop)
└── Interaction layer (pan, zoom, click-to-navigate)
```

**Data source:**
- On panel activation, fetch the full link graph:
  ```ts
  // New Tauri command needed
  const graph = await getFullLinkGraph(workspace);
  // Returns: { nodes: [{id, label}], edges: [{source, target}] }
  ```

**Force-directed layout:**
- Implement a simple force simulation (or use `d3-force` if bundle size is acceptable):
  - **Repulsion**: nodes repel each other (charge force).
  - **Attraction**: linked nodes attract (spring force along edges).
  - **Centering**: nodes gravitate toward canvas center.
  - **Damping**: simulation settles after ~2 seconds.
- Run the simulation in `requestAnimationFrame` loop.
- Cool down gradually — stop the loop when motion is below threshold.

**Rendering (Canvas 2D):**
- **Nodes**: Small circles (r=6px).
  - Default: `vigil-text-muted` fill.
  - Active file: `vigil-teal` fill, larger (r=10px), with a subtle glow.
  - Hovered: `vigil-text` fill, slight scale-up.
- **Edges**: Thin lines (1px) in `vigil-border` color. Subtle opacity.
- **Labels**: File name (without `.md`) rendered below each node.
  - Only show labels for nodes within a zoom threshold (declutter at zoom-out).
  - Font: 10px, `vigil-text-muted`.

**Interactions:**
- **Pan**: Click + drag on empty space.
- **Zoom**: Scroll wheel. Clamp between 0.2x and 5x.
- **Click node**: Navigate to that file (set `editorState.activeFile`).
- **Hover node**: Highlight the node and all its direct connections.

### `src-tauri/src/commands/search.rs` (extend)

Add Tauri command:

```rust
#[derive(Serialize)]
pub struct GraphNode {
    pub id: String,     // file path
    pub label: String,  // file name without extension
}

#[derive(Serialize)]
pub struct GraphEdge {
    pub source: String, // source file path
    pub target: String, // target file path
}

#[derive(Serialize)]
pub struct LinkGraphData {
    pub nodes: Vec<GraphNode>,
    pub edges: Vec<GraphEdge>,
}

#[tauri::command]
pub fn get_full_link_graph(workspace: String) -> Result<LinkGraphData, String>
```

This reads from the existing `LinkGraph` (Task 1.7) and serializes its full
state as nodes + edges.

## Styling

Container:
```
w-full h-full
bg-vigil-bg
overflow-hidden
```

The graph canvas fills the entire sidebar panel area. No scrollbars.

## Performance Targets

- Render up to **1,000 nodes** at 60fps.
- Initial layout settles in < 2 seconds.
- Pan/zoom is instant (no lag).

## Acceptance Criteria

- [ ] Graph panel is accessible via the Graph icon in the Activity Bar
- [ ] Nodes represent `.md` files, edges represent `[[wikilink]]` connections
- [ ] The currently active file is highlighted in teal
- [ ] Clicking a node navigates to that file in the editor
- [ ] Hovering a node highlights its direct connections
- [ ] Pan and zoom work smoothly
- [ ] Graph handles 500+ notes without visible lag
- [ ] Orphan notes (no links) appear as disconnected nodes
- [ ] The graph updates when links are added or removed (on file save)
